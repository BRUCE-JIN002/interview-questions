# 小程序平台能力与合规经验

## 平台约束意识

微信小程序不是普通 H5 容器。实际工作中必须关注：

- 请求域名白名单。
- HTTPS/WSS 要求。
- 限频 API。
- 隐私接口授权。
- 订阅消息主开关和模板状态。
- 分包大小和主包体积。
- 原生组件路径和 usingComponents。
- Canvas 图片格式限制。

## 域名白名单

新增请求域名时，小程序后台必须配置服务器域名，否则请求会被宿主层拦截。

经验：

- 新增接口域名时先查是否已有。
- 不能使用未配置域名、IP、非 HTTPS、带端口的地址。
- H5 能请求成功不代表小程序能请求成功。

## 限频 API

微信对一些接口有限频要求：

- `wx.login`
- `wx.checkSession`
- `wx.getSetting`
- `wx.getUserInfo`
- `wx.getUserProfile`

项目里的体现：

- `syncWeappIdentity` 用 promise 去重。
- 订阅消息设置不是每次渲染都查，而是有初始化状态控制。
- 订阅流程中查询模板 itemSetting，避免无意义拉起。

经验：这些 API 不要放在循环、滚动、无依赖 `useEffect`、频繁 onShow 中反复调用。

## 订阅消息

项目订阅消息流程包括：

- 初始化主开关状态。
- 查询场景订阅状态。
- 点击时先检查主开关。
- 检查模板是否被永久拒绝。
- 发起 `Taro.requestSubscribeMessage`。
- 上报授权结果。
- 若用户勾选“总是保持以上选择”并拒绝，引导去设置页。

这比简单调用 `requestSubscribeMessage` 完整得多。

## 隐私合规

使用位置、相机、通讯录、剪贴板、录音、用户信息等隐私接口前，需要确认：

- `app.json` 声明隐私版本。
- 调用前已弹出隐私授权。
- 授权弹窗是可交互 UI。

否则微信可能返回错误码 103。

## 小程序 Canvas 资源

iOS 微信小程序 Canvas 不支持 SVG。参与截图的资源必须用 PNG/JPG。

另外：

- 网络图片要可访问。
- CDN 图片要考虑跨域和格式。
- 小字号会影响截图清晰度。
- 原生组件资源要正确 copy 到 dist。

## 小程序字体

部分 Android 微信小程序对小于 10px 的字号渲染异常。规则：

- 不直接写小于 10px 的 `font-size`。
- 需要小字号视觉时用 `font-size: 10px` + `transform: scale()`。
- 参与海报截图的文字尤其要规避。

## 面试表达

> 我做小程序不会只关注 Taro 代码，还会关注微信平台约束。比如新增域名要配置白名单，`wx.login/getSetting/requestSubscribeMessage` 这类能力要控制调用频率，订阅消息要处理主开关、模板永久拒绝和授权结果上报。隐私接口要先完成微信隐私授权流程。Canvas 截图里 iOS 小程序不支持 SVG，所以截图资源要用 PNG/JPG。
