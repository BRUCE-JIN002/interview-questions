# 2026-07-24 全栈偏前端每日面试题

## 1. JavaScript：Top-level `await`、模块图与循环依赖

- 题型：原理 + 排障
- 难度：较难
- 可能来源或考察方向：[MDN top-level await](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Operators/await#top_level_await)
- 题目：ESM 中模块 A 顶层 `await` 配置，模块 B 导入 A，A 又通过模块 C 间接导入 B。说明顶层 `await` 如何影响模块求值顺序；它与普通异步函数有什么不同？如何定位启动卡住或访问到未初始化导出的循环依赖，并给出避免方案？

## 2. TypeScript：声明合并、模块扩展与类型污染边界

- 题型：类型设计 + 工程实践
- 难度：中等
- 可能来源或考察方向：[TypeScript Declaration Merging](https://www.typescriptlang.org/docs/handbook/declaration-merging.html)
- 题目：项目要为第三方请求库扩展 `RequestConfig`，加入 `traceId`，同时不能让全局 `Window` 或第三方模块类型被意外污染。如何区分全局声明、模块扩展与本地包装类型？扩展文件怎样写才会被 TypeScript 正确识别？为什么运行时仍需保证字段真正被传递？

## 3. ES6+：`using`、`Symbol.dispose` 与异步资源释放

- 题型：原理 + 场景
- 难度：较难
- 可能来源或考察方向：[TC39 Explicit Resource Management](https://github.com/tc39/proposal-explicit-resource-management)
- 题目：服务端脚本要临时打开文件、数据库游标和锁；异常路径曾遗漏 `finally`。解释 `using` / `await using`、`Symbol.dispose` / `Symbol.asyncDispose` 的释放时机和 LIFO 顺序。若目标运行时尚未支持，该如何用 `try/finally` 保持等价的资源所有权模型？

## 4. 浏览器性能：Long Task、INP 与 `scheduler.postTask`

- 题型：性能排障 + 场景设计
- 难度：较难
- 可能来源或考察方向：[web.dev INP](https://web.dev/articles/inp)；[Prioritized Task Scheduling API](https://developer.mozilla.org/docs/Web/API/Prioritized_Task_Scheduling_API)
- 题目：运营后台点击筛选后主线程连续卡顿 300ms，接口其实很快。如何用 Performance 面板和 Long Task 归因到 JS、样式计算或第三方脚本？怎样将可切分工作安排到 `scheduler.postTask`、`requestAnimationFrame`、Web Worker 或空闲期，并避免“拆分后仍阻塞关键交互”？

## 5. 网络：Range 请求、断点续传与内容完整性

- 题型：接口设计 + 场景
- 难度：中等
- 可能来源或考察方向：[MDN HTTP Range requests](https://developer.mozilla.org/docs/Web/HTTP/Guides/Range_requests)
- 题目：前端下载一个数 GB 的导出文件，弱网下希望暂停并恢复。说明 `Range`、`Accept-Ranges`、`206 Partial Content`、`Content-Range` 与 `ETag` 的协作方式；服务端文件更新、CDN 缓存和多段响应时如何避免把不同版本的字节拼在一起？

## 6. React：Strict Mode 下 Effect 的重复执行与外部系统同步

- 题型：原理 + 排障
- 难度：中等
- 可能来源或考察方向：[React useEffect](https://react.dev/reference/react/useEffect#my-effect-runs-twice-when-the-component-mounts)
- 题目：开发环境打开 Strict Mode 后，聊天室出现两次连接、埋点重复上报。React 为什么会额外执行一次 setup/cleanup？如何把 Effect 设计成可重入的“连接外部系统”同步，而不是用 ref 标记跳过？哪些动作应放在事件处理器而非 Effect？

## 7. Vue 3：错误捕获边界与异步组件的降级策略

- 题型：场景设计
- 难度：中等
- 可能来源或考察方向：[Vue errorCaptured](https://vuejs.org/api/composition-api-lifecycle.html#onerrorcaptured)
- 题目：低频报表模块按需加载，网络失败或渲染异常不能使整个后台白屏。如何组合 `defineAsyncComponent` 的 loading/error/timeout、组件 `onError`、`onErrorCaptured` 与全局 `app.config.errorHandler`？错误上报应包含哪些上下文，又怎样避免错误边界自身无限重试？

## 8. 手写：带 TTL 和 O(1) 淘汰的 LRU 缓存

- 题型：手写实现
- 难度：较难
- 可能来源或考察方向：综合整理
- 题目：实现 `LRUCache`，支持 `get(key)`、`set(key, value, ttlMs?)`；容量满时淘汰最久未使用项，过期项不可返回。要求平均 O(1)，说明哈希表与双向链表各负责什么；在 Node 多请求共享缓存中，TTL、缓存空值和缓存击穿分别如何处理？

## 9. 算法：数据流中的第 K 大元素

- 题型：算法题 + 手写
- 难度：中等
- 可能来源或考察方向：[LeetCode 703](https://leetcode.com/problems/kth-largest-element-in-a-stream/)
- 题目：设计 `KthLargest(k, nums)` 与 `add(val)`，每次加入后返回当前数据流第 k 大元素。给出最小堆解法及复杂度；为什么维护大小为 k 的最小堆比每次全量排序更合适？若需要支持删除历史元素，数据结构如何调整？

## 10. Next.js：客户端组件边界与第三方浏览器 SDK

- 题型：框架工程 + 排障
- 难度：中等
- 可能来源或考察方向：[Next.js Client Components](https://nextjs.org/docs/app/getting-started/server-and-client-components)
- 题目：在 App Router 中接入只支持浏览器的地图 SDK，直接导入后 SSR 报 `window is not defined`，并且页面 bundle 明显变大。如何划分 Server/Client Component 边界、按需加载 SDK、在卸载时清理实例，并把服务端获取的可序列化数据传给客户端？为什么不能简单把整个 page 标记为 `'use client'`？

## 11. Node.js：HTTP 流式上传的背压、大小限制与中止

- 题型：系统设计 + 排障
- 难度：较难
- 可能来源或考察方向：[Node.js Streams](https://nodejs.org/api/stream.html)
- 题目：BFF 接收大文件后转发到对象存储，线上出现内存飙升和客户端断开后仍持续上传。如何用 stream pipeline 而不是把 body 聚合成 Buffer？如何设置大小/类型限制、传播 `AbortSignal`、处理背压与失败清理，并避免把不可信文件名直接用于对象 key？

## 12. NestJS：API 版本演进与兼容性治理

- 题型：架构设计
- 难度：中等
- 可能来源或考察方向：[NestJS Versioning](https://docs.nestjs.com/techniques/versioning)
- 题目：移动端与 Web 端发布节奏不同，订单接口字段需要重命名且旧客户端至少再支持 90 天。NestJS 中如何选择 URI、Header 或媒体类型版本控制？如何让 DTO、响应序列化、OpenAPI 文档、灰度与下线公告同步演进，避免仅靠“字段可选”悄悄破坏语义？

## 13. 数据库与接口：游标分页的一致性与翻页令牌安全

- 题型：系统设计
- 难度：较难
- 可能来源或考察方向：综合整理
- 题目：动态订单列表使用 offset 分页时，新增订单导致重复或漏数据。设计基于 `(created_at, id)` 的游标分页：SQL 条件、升降序、下一页 token 编码和索引该怎么做？游标中哪些信息需要签名或绑定筛选条件，为什么不能让客户端随意篡改？

## 14. AI 前端：生成式 UI 的结构化输出与安全降级

- 题型：AI 前端场景设计
- 难度：较难
- 可能来源或考察方向：[OpenAI Structured Outputs](https://platform.openai.com/docs/guides/structured-outputs)
- 题目：客服 Agent 根据对话生成“退款进度卡片”，前端不能直接渲染模型返回的任意 HTML。如何用 JSON Schema/结构化输出定义可渲染组件、字段校验与版本兼容？流式过程中 JSON 尚不完整时如何展示，校验失败时怎样降级为安全的纯文本并保留可观测信息？

## 15. Agent/RAG：多租户检索中的权限过滤与可解释引用

- 题型：Agent 架构设计
- 难度：较难
- 可能来源或考察方向：综合整理
- 题目：企业 Agent 为多个租户检索知识库；向量召回速度优先，但绝不能越权引用。ACL 过滤应在索引、检索、重排、生成和前端展示的哪些阶段执行？如何将文档版本、片段 id、授权结果写入引用链路，并在用户权限变化后处理缓存、会话记忆和已生成答案？
