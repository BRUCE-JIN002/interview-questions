# 2026-07-09 全栈偏前端每日面试题

## 1. 微任务过多导致页面卡死如何定位和治理

- 题型：八股文 + 场景题
- 难度：中高级
- 可能来源或考察方向：综合整理；JavaScript 事件循环、浏览器渲染、性能治理
- 题目正文：线上页面点击按钮后没有长任务报警，但输入、滚动和动画明显卡顿。排查发现某段逻辑不断通过 `Promise.resolve().then()` 递归调度微任务。请说明微任务、宏任务、渲染之间的关系，为什么微任务链可能饿死渲染，并设计治理方案。

## 2. TypeScript 中 `keyof`、索引访问类型与映射类型的组合建模

- 题型：类型设计题
- 难度：中级
- 可能来源或考察方向：综合整理；TypeScript 类型系统、接口客户端建模
- 题目正文：给定一组接口定义 `{ user: { req; res }, order: { req; res } }`，请设计类型工具，让 `request("user", payload)` 能自动推导对应的请求参数和返回类型。要求说明 `keyof`、`T[K]`、映射类型各自承担的角色。

## 3. 手写支持最大重试次数、指数退避和抖动的请求重试函数

- 题型：手写实现题
- 难度：中高级
- 可能来源或考察方向：综合整理；Promise、网络容错、前端稳定性
- 题目正文：实现 `retry(fn, options)`，其中 `fn` 返回 Promise，`options` 支持 `retries`、`baseDelay`、`maxDelay`、`shouldRetry`。要求使用指数退避和随机抖动，避免所有客户端同时重试，并说明哪些错误不应该重试。

## 4. CSS Container Queries 与响应式组件设计

- 题型：场景题
- 难度：中级
- 可能来源或考察方向：MDN Container Queries；组件级响应式、设计系统
- 题目正文：同一个商品卡片会出现在首页瀑布流、侧边栏和弹窗中。传统媒体查询只能根据视口宽度响应，导致组件复用困难。请说明 Container Queries 的工作方式、适用场景、潜在成本，以及如何设计可复用卡片组件。

## 5. React Context 频繁更新引发大面积重渲染

- 题型：框架原理 + 场景题
- 难度：中高级
- 可能来源或考察方向：综合整理；React 性能优化、状态管理边界
- 题目正文：一个后台页面把用户信息、权限、主题、筛选条件都放在同一个 Context 中，筛选条件变化时整个页面大量组件重渲染。请分析原因，并给出不引入重型状态库时的拆分、选择器、外部 store 或 memo 化方案。

## 6. Vue 3 `flush` 时机与 DOM 读取一致性

- 题型：框架原理题
- 难度：中高级
- 可能来源或考察方向：Vue 官方文档；`watch` / `watchEffect`、更新队列、DOM 读取
- 题目正文：在 Vue 3 中监听响应式状态后立即读取 DOM 尺寸，有时读到的是旧值。请解释 `flush: "pre"`、`flush: "post"`、`flush: "sync"` 的差异，以及在表单校验、动画测量和日志埋点中分别如何选择。

## 7. Fetch Priority 与关键资源加载优化

- 题型：浏览器网络题
- 难度：中级
- 可能来源或考察方向：web.dev Fetch Priority；LCP 优化、资源优先级
- 题目正文：页面首屏 LCP 图片被浏览器较晚发现，导致加载瀑布中优先级偏低。请说明 `fetchpriority`、`preload`、懒加载和资源发现顺序的关系，并设计一套不破坏带宽公平性的首屏资源优化方案。

## 8. 手写事件总线的 once、off 与异常隔离

- 题型：手写实现题
- 难度：中级
- 可能来源或考察方向：综合整理；发布订阅、组件通信、异常边界
- 题目正文：实现一个 `EventBus`，支持 `on`、`once`、`off`、`emit`。要求同一个事件多个监听器互不影响，一个监听器抛错不能阻断其他监听器，并说明如何避免 `once` 包装函数导致 `off` 失效。

## 9. 算法：两个有序数组的中位数

- 题型：算法题
- 难度：困难
- 可能来源或考察方向：LeetCode 4；二分、边界处理、复杂度分析
- 题目正文：给定两个升序数组 `nums1` 和 `nums2`，返回两个数组合并后的中位数。要求时间复杂度达到 `O(log(m+n))`，并说明分割线、奇偶长度、空数组和边界哨兵的处理。

## 10. Next.js Route Segment Config 与动态渲染边界

- 难度：中高级

- 题目正文：一个详情页既有可缓存的商品信息，也有依赖登录态的价格和库存提示。请说明 `dynamic`、`revalidate`、`fetchCache`、`runtime` 等 Route Segment Config 的作用，并设计静态内容与用户态内容的拆分方案。
