# 2026-07-17 全栈偏前端每日面试参考答案

## 1. JavaScript：`Object.hasOwn`、原型链与原型污染

- 参考答案：`obj.hasOwnProperty` 可被同名自有属性覆盖，且 `Object.create(null)` 没有该方法；用 `Object.hasOwn(obj, key)`。`in` 会查询原型链。外部对象不要无约束深合并到普通对象；拒绝 `__proto__`、`constructor`、`prototype` 等键，使用白名单/schema 校验，必要时用无原型字典。
- 关键考点：自有属性；原型链；不可信输入。
- 常见追问或变体：`JSON.parse` 本身不会修改原型，危险通常发生在后续赋值或深合并。

## 2. TypeScript：用 `infer` 解析查询参数类型

- 参考答案：先递归按 `&` 切分，再按 `=` 用 `infer` 提取键值；同键时用条件类型把旧值提升为元组并追加新值。类型只描述字面量，真实 URL 仍需运行时解析和校验；长字符串可能碰到编译器递归限制。
- 关键考点：模板字面量类型；`infer`；递归条件类型；类型/运行时边界。
- 常见追问或变体：处理无值参数、URL 解码、键顺序与数组策略。

## 3. ES6+：异步迭代器的错误、取消与资源回收

- 参考答案：循环 `break` 会尝试调用迭代器 `return()`；循环体抛错也会进行 IteratorClose；底层 IO 还应监听 `AbortSignal` 主动取消。生成器内使用 `try/finally` 释放 reader、计时器或连接，`return()` 中转发 abort。
- 关键考点：AsyncIterator 协议；`return()`；`finally`；取消不是自动发生的。
- 常见追问或变体：`throw()` 由消费者显式调用；已入队的数据是否继续消费要在契约中明确。

## 4. 浏览器：用 PerformanceObserver 建立真实用户性能监控

- 参考答案：用 `PerformanceObserver` 订阅 paint、largest-contentful-paint、layout-shift、event、longtask、resource 等 entry，并附带路由、设备、网络和发布版本。按会话/用户采样，聚合后上报；不要传完整 URL query、DOM 文本、输入内容或可识别资源名。交互归因用自定义 mark/measure 与 trace ID 关联。
- 关键考点：Web Vitals；buffered；采样；数据最小化。
- 常见追问或变体：CLS 排除最近用户输入后的位移；LCP 在页面隐藏后最终确定。

## 5. 网络：Range 请求与媒体/大文件断点下载

- 参考答案：客户端发 `Range: bytes=start-end`，服务端返回 `206`、`Content-Range: bytes start-end/total`，`Accept-Ranges: bytes` 表示支持范围请求。`If-Range` 搭配 ETag/日期，资源变更则回退完整响应。CDN 必须正确缓存分片；内容压缩会改变字节偏移，下载通常禁用动态压缩；签名 URL 需覆盖分片请求有效期。
- 关键考点：部分内容响应；ETag；CDN；字节语义。
- 常见追问或变体：服务端忽略 Range 时可能返回 `200`，客户端要降级处理。

## 6. React：错误边界、异步错误与恢复策略

- 参考答案：错误边界捕获后代渲染、生命周期和构造期错误；不捕获事件处理器、异步回调、服务端渲染及边界自身错误。路由级边界负责整页兜底，模块级边界负责局部恢复；事件/请求用 `try/catch` 后写入可渲染错误状态。日志带 route、release、request/trace ID，重试需幂等并防止循环。
- 关键考点：捕获范围；分层降级；错误关联。
- 常见追问或变体：Suspense 处理等待，不等同于错误边界。

## 7. Vue 3：Teleport 的层级、焦点与 SSR 一致性

- 参考答案：定义统一 overlay 层级表和栈管理器；打开时保存触发元素、把焦点移入弹窗并 trap，关闭时恢复；仅顶层响应 ESC；引用计数管理 body 滚动锁。SSR 模板应包含稳定的目标节点，客户端 target 与服务端一致，避免 hydration mismatch。
- 关键考点：可访问性；overlay stack；SSR。
- 常见追问或变体：多个 Teleport 到同一 target 的顺序依赖挂载顺序，不应成为业务层级依据。

## 8. 手写：支持占位符的 `curry`

- 参考答案：维护已收集参数；新参数优先填充旧数组中的占位符，剩余参数追加；非占位实参数达到 `fn.length` 即 `fn.apply(lastThis, args)`，多出的参数原样传入。空调用可返回自身；最后一跳保存 `this`。

```js
const curry = (fn, ph = curry._) => {
  const build = collected => function curried(...incoming) {
    const args = [...collected];
    let i = 0;
    for (let p = 0; p < args.length && i < incoming.length; p++) {
      if (args[p] === ph) args[p] = incoming[i++];
    }
    args.push(...incoming.slice(i));
    if (args.filter(x => x !== ph).length >= fn.length) return fn.apply(this, args);
    return build(args);
  };
  return build([]);
};
curry._ = Symbol('placeholder');
```

- 关键考点：占位符补位；`fn.length`；`this` 语义。
- 常见追问或变体：生产实现应避免把业务 `this` 与内部存储对象混用，可用闭包保存参数和最后上下文。

## 9. 算法：最小化最大值的二分答案

- 参考答案：答案范围是 `[max(nums), sum(nums)]`。给定上限 `mid`，贪心累加；超过 `mid` 就新开一段。若所需段数 `<= m`，说明上限可行并收缩右边界，否则增大左边界。上限越大，所需段数不会增加，因此具有单调性。
- 关键考点：二分答案；贪心判定；单调性。
- 常见追问或变体：数组含负数时该贪心及单调判定不再直接适用。

## 10. Next.js：异步 Request API 迁移与动态边界

- 参考答案：检索所有 `cookies`、`headers`、`params`、`searchParams` 访问点，按调用链改为 `await`，客户端读取 params 时使用 `React.use` 等对应方式。它们依赖请求时值，读取会令所在路由/段动态。把仅与请求相关的读取下沉到最小 Server Component 或 Route Handler，静态布局不要透传 request 对象。
- 关键考点：动态 API；App Router；静态/动态边界。
- 常见追问或变体：缓存策略要和数据访问位置一起审查，不能只靠页面级开关。

## 11. Node.js：Undici/Fetch 连接池与上游隔离

- 参考答案：为稳定上游复用 dispatcher/pool，配置连接数和连接生命周期；每个上游独立并发阈值与熔断/降级，调用采用总预算加子请求预算并传递 `AbortSignal`。每请求创建客户端会失去 keep-alive、造成握手和 socket 抖动。错误分类记录 DNS、连接、首字节和响应阶段。
- 关键考点：连接复用；bulkhead；超时预算；取消。
- 常见追问或变体：连接池不是无限并发许可，排队时间也应纳入超时预算。

## 12. NestJS：DTO 泛型与运行时元数据边界

- 参考答案：TS 泛型在编译后擦除，运行时反射不知道 `T` 的具体类，Swagger 和 transformer 无法自动生成正确 items schema。可用 DTO 工厂：传入实体 class，返回带 `@ApiExtraModels`、`@ApiProperty({ type: () => Item })` 和 `@Type(() => Item)` 的具体 Page DTO；controller 使用该具体类。
- 关键考点：类型擦除；反射元数据；OpenAPI 与校验一致。
- 常见追问或变体：不要只为文档写一个类型、运行时另用一个对象，避免契约漂移。

## 13. 数据库：零停机 Schema 迁移的 Expand-Contract

- 参考答案：先 Expand：新增可空列/索引，部署可同时读新旧列的代码；回填历史数据并持续校验；短期双写或由单一写路径同步，读优先新列并回退旧列。确认旧版本和消费者下线后，设约束、切断旧写、最终删除旧列（Contract）。大表索引采用在线能力，迁移前后监控锁等待、缺失率和差异率；每阶段都可切回旧读写。
- 关键考点：兼容窗口；回填；双写风险；可观测回滚。
- 常见追问或变体：双写要接受最终一致或采用 outbox/单写者，不能假设跨服务事务。

## 14. AI 前端：浏览器操作 Agent 的可确认执行界面

- 参考答案：把计划、工具调用和执行结果显式分开。填表只生成草稿并允许逐字段编辑；提交前展示目标站点、字段 diff 和不可逆影响，要求明确确认。对可逆操作提供撤销或补偿；页面内容一律是不可信数据，不能变成指令。操作记录应关联用户、会话、授权范围和证据快照。
- 关键考点：human-in-the-loop；最小授权；提示注入；可逆性。
- 常见追问或变体：确认不应是泛化的“一律允许”，应绑定具体操作和短时权限。

## 15. Agent：工具调用的审计事件模型与可观测性

- 参考答案：每次调用记录 `traceId/spanId`、actor/session、授权 scope、工具名与版本、输入/输出的 schema 化摘要或哈希、开始结束时间、结果、错误分类、审批 ID、幂等键和副作用资源 ID。原文按敏感字段掩码、最短保留期和访问控制存储；需要回放时保存脱敏快照和工具版本，而非默认保存全部 prompt/响应。
- 关键考点：关联性；可复现性；数据最小化；审计不可篡改。
- 常见追问或变体：将高风险写操作与只读调用分级，分别设置保留期和告警规则。
