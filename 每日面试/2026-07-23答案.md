# 2026-07-23 全栈偏前端每日面试参考答案

## 1. JavaScript：`AbortSignal` 的组合、超时与取消语义

**参考答案：**每次搜索创建本次 controller；用 `AbortSignal.any([controller.signal, AbortSignal.timeout(5000)])` 合并新输入、卸载和超时。降级时监听多个 signal，任一 abort 就调用自己的 controller.abort(reason)。`fetch` 被取消只表示客户端不再等待/读取；服务端可能已执行，应对写操作另设幂等与取消协议。

**关键考点：**取消原因、组件 cleanup、超时不等于业务事务回滚。

**常见追问或变体：**不要复用已 abort 的 signal；捕获 `AbortError` 时不要误报为业务失败。

## 2. TypeScript：`const` 类型参数与字面量推断

**参考答案：**`const T` 要求推断尽量保留对象/数组的字面量与只读结构，减少调用方手写 `as const`。例如：

```ts
function defineRoutes<const T extends readonly { path: `/${string}` }[]>(routes: T) {
  return routes;
}
const routes = defineRoutes([{ path: '/' }, { path: '/settings' }]);
type Path = (typeof routes)[number]['path']; // '/' | '/settings'
```

它只影响编译期推断；路由参数、权限和外部配置仍须 schema 校验。

**关键考点：**字面量保留、泛型约束与运行时边界。

**常见追问或变体：**`as const` 可用于任意表达式；`const` 类型参数由 API 作者在泛型声明处提供体验。

## 3. ES6+：迭代器提前退出与资源清理

**参考答案：**在 generator 的 `try/finally` 内关闭订阅、终止请求；`for...of` 的 `break`、异常和显式调用迭代器 `return()` 都会触发 `finally`。异步 generator 同理，`for await...of` 退出会等待其 `return()` 完成，因此清理可 `await`。

**关键考点：**迭代协议的 `return()`、`finally` 可靠收尾、主动取消上游。

**常见追问或变体：**若消费者根本不开始迭代，generator 体不会执行，也就不存在其中创建的资源。

## 4. 浏览器：Shadow DOM 事件委托与跨边界事件

**参考答案：**为封装性，跨 shadow 边界后 `event.target` 会重定向到宿主；`composedPath()` 可查看实际传播路径（closed root 对外仍受限制）。自定义事件只有 `composed: true` 才能越过边界，通常还需 `bubbles: true`。组件应向外派发语义化事件和必要 payload，而非要求外部依赖内部按钮节点。

**关键考点：**事件重定向、封装边界、自定义事件契约。

**常见追问或变体：**不要因方便把 closed shadow root 改 open；测试应覆盖宿主监听与键盘触发。

## 5. 网络：Service Worker 缓存与“离线可用但数据不陈旧”

**参考答案：**静态不可变资源可 Cache First；时效敏感列表常用 Network First 并设短超时；允许短暂旧内容的公共列表可 Stale While Revalidate。缓存 key 必须包含语言、版本和无敏感身份的变体；私有响应通常不进共享 Cache Storage，登出时删除用户 scoped cache。新 worker 通过版本化 cache、`activate` 清理旧版本，并以服务端版本号/推送通知触发重拉。

**关键考点：**缓存策略按数据风险选择，离线缓存不是权限系统。

**常见追问或变体：**不要缓存带 `Set-Cookie` 或用户专属响应；离线时应明确展示数据更新时间。

## 6. React：受控输入、延迟值与昂贵列表渲染

**参考答案：**input 的 state 必须同步更新，否则会出现输入滞后。把原始 query 立即写入 state，再把 `useDeferredValue(query)` 传给昂贵过滤列表；稳定 props 配合 `memo`，大数据再虚拟化。延迟值是在渲染优先级上允许旧列表暂留，防抖是合并时间窗口内的副作用，常用于网络请求，二者可配合但不可互相替代。

**关键考点：**输入紧急更新、派生 UI 延迟、渲染优化与请求节流分层。

**常见追问或变体：**以 deferred query 标记列表 stale，不能把它当作服务端查询的一致性保证。

## 7. Vue 3：`shallowRef`、不可变数据与大对象更新

**参考答案：**`shallowRef` 只跟踪 `.value` 替换，适合第三方实例和整体替换的大树；修改其内部可变对象后可调用 `triggerRef`。`markRaw` 用于永不应代理的地图/图表实例。不可变更新应创建新根引用并赋给 `shallowRef.value`，让 Vue 以一次顶层变更更新视图。

**关键考点：**深代理成本、第三方对象身份、顶层替换。

**常见追问或变体：**`triggerRef` 是显式通知，不应把它当作频繁深层突变的常规方案。

## 8. 手写：并发受限、可取消的异步任务池

**参考答案：**用下一个索引、结果数组和运行计数派发任务；每次完成后仅在未失败/未取消时补位。取消时停止派发并 reject，已开始的任务需要自身接收 signal 才能真正中止。

```ts
async function runPool<T>(tasks: Array<(signal: AbortSignal) => Promise<T>>, options: { concurrency: number; signal?: AbortSignal }) {
  const controller = new AbortController();
  options.signal?.addEventListener('abort', () => controller.abort(), { once: true });
  const result: T[] = Array(tasks.length);
  let next = 0;
  async function worker() {
    while (!controller.signal.aborted) {
      const i = next++;
      if (i >= tasks.length) return;
      try { result[i] = await tasks[i](controller.signal); }
      catch (error) { controller.abort(); throw error; }
    }
  }
  await Promise.all(Array.from({ length: Math.min(options.concurrency, tasks.length) }, worker));
  if (controller.signal.aborted) throw controller.signal.reason ?? new DOMException('Aborted', 'AbortError');
  return result;
}
```

**关键考点：**最大并发、顺序结果、失败停止派发、协作式取消。

**常见追问或变体：**生产版要校验 `concurrency > 0`，并区分外部取消与任务失败原因。

## 9. 算法：滑动窗口的最短子数组

**参考答案：**正整数使窗口和随右指针右移单调不减；达到 target 后不断移动左指针缩小窗口并记录最小值，因此每个元素最多进出窗口一次。

```ts
function minSubArrayLen(target: number, nums: number[]) {
  let left = 0, sum = 0, best = Infinity;
  for (let right = 0; right < nums.length; right++) {
    sum += nums[right];
    while (sum >= target) {
      best = Math.min(best, right - left + 1);
      sum -= nums[left++];
    }
  }
  return best === Infinity ? 0 : best;
}
```

负数会破坏单调性，左移可能让之后的更长窗口重新满足条件，需要前缀和加单调队列等不同方法。

**关键考点：**正数前提、双指针 O(n)、负数边界。

**常见追问或变体：**若只求是否存在，仍可滑窗；若求固定和且含负数，可用前缀和哈希。

## 10. Next.js：动态路由参数、404 与缓存边界

**参考答案：**`generateStaticParams` 只预生成热门 slug；页面在服务端读取权威商品数据，不存在或下架即调用 `notFound()`，从而返回正确状态码和可索引的 404 页面。修改/下架后按商品 tag 或路径 revalidate。客户端空态跳转会先输出错误的成功页，SEO、状态码和闪烁都不正确。

**关键考点：**服务器决定资源存在性、预渲染不是授权、按资源失效。

**常见追问或变体：**个性化价格和库存不应误混进可共享的静态缓存。

## 11. Node.js：`EventEmitter` 监听器泄漏与一次性事件

**参考答案：**先用 heap snapshot、`listenerCount()` 和调用栈定位谁在每次请求重复注册；请求结束、超时和 abort 都要解除监听。只等待一次结果用 `once()`；能与取消绑定的场景用 `addAbortListener` 或自行在 abort 时 `off()`。提高 `setMaxListeners` 只是确认确有合理高扇出后的阈值调整，不能掩盖泄漏。

**关键考点：**Emitter 生命周期、监听器对闭包的强引用、取消清理。

**常见追问或变体：**匿名回调难以 `off`，应保存函数引用或使用一次性监听。

## 12. NestJS：动态模块与可测试的第三方客户端封装

**参考答案：**模块以 `forRootAsync` 注入经过 schema 校验的配置，暴露稳定 token（如 `PAYMENT_CLIENT`）和业务接口；provider 内根据供应商实例化 SDK，业务 Service 只注入接口 token。连接初始化/销毁放 provider 生命周期钩子，密钥轮换通过受控配置刷新或重建客户端；测试以同 token 覆盖 fake provider。

**关键考点：**依赖倒置、异步配置、生命周期与 mock 替换。

**常见追问或变体：**不要在每个请求里构造 SDK；敏感配置不可写入日志或异常响应。

## 13. 接口设计：Webhook 签名校验与至少一次投递

**参考答案：**服务端保存原始 body，用共享密钥对时间戳与 body 做 HMAC 并常量时间比较；拒绝过期时间戳。以供应商 event id 做唯一去重，先持久化接收记录再异步处理，状态迁移必须校验合法前序，乱序事件可暂存/查询权威状态。浏览器无法保守密钥、无法稳定公网接收且生命周期短，不能作为 webhook 端点。

**关键考点：**原始 body、重放窗口、去重、最终以服务端状态机收敛。

**常见追问或变体：**不能仅靠订单号去重；不同事件类型或供应商需要清晰的去重作用域。

## 14. AI 前端：流式回答的敏感信息脱敏与权限提示

**参考答案：**检索阶段按 ACL 过滤；送模型前最小化字段并脱敏；输出流在 BFF 逐 chunk 扫描/按结构化事件审核，高风险命中应先缓冲到可判断边界再展示，不能依赖“展示后撤回”。日志用独立访问控制、哈希/掩码和短保留期。前端展示可访问的引用，权限不足或被策略拦截时给出泛化原因，不泄露文档标题和存在性。

**关键考点：**授权先于生成、流式拦截窗口、日志同样是数据面。

**常见追问或变体：**正则脱敏只能作一层，密钥/业务数据还应分类、DLP 与人工流程兜底。

## 15. Agent：规划与执行分离下的失败恢复

**参考答案：**计划只包含目标、候选步骤、参数摘要和风险，先供用户/策略引擎审批；执行时为每步记录 execution id、输入快照、工具版本、状态和幂等键。查询可重试，草稿创建用幂等键，发布必须二次确认并在恢复前按幂等键对账。计划改变应生成新版本，旧计划不可静默继续执行。

**关键考点：**意图与授权分离、checkpoint、幂等、恢复前对账。

**常见追问或变体：**工具结果和模型文本都不是授权凭据；高风险步骤应有最小权限与审计事件。
