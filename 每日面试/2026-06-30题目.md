# 2026-06-30 全栈偏前端面试题

> 分布：前端核心 9 题、全栈工程 4 题、AI / Agent 2 题。

## 1. JavaScript 显式资源管理与异步释放

- 题型：八股文 / 场景题
- 难度：中高级
- 来源或考察方向：[TC39 Explicit Resource Management](https://github.com/tc39/proposal-explicit-resource-management)
- 题目：解释 `using`、`await using`、`Symbol.dispose` 和 `Symbol.asyncDispose` 的作用。与 `try...finally` 相比，它们解决了什么问题？在前端或 Node.js 中分别举一个适用场景，并说明资源释放的顺序和异常处理方式。

## 2. TypeScript 函数类型的协变、逆变与双向协变

- 题型：类型系统 / 八股文
- 难度：高级
- 来源或考察方向：[TypeScript strictFunctionTypes](https://www.typescriptlang.org/tsconfig/strictFunctionTypes.html)
- 题目：解释函数参数和返回值的型变规则。为什么开启 `strictFunctionTypes` 后，`(dog: Dog) => void` 不能赋给 `(animal: Animal) => void`？为什么方法声明又可能表现不同？请结合事件回调 API 说明错误建模可能造成的运行时问题。

## 3. 手写 `Promise.any`

- 题型：手写实现
- 难度：中级
- 来源或考察方向：[MDN Promise.any](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Promise/any)
- 题目：不调用原生 `Promise.any`，实现 `promiseAny(iterable)`。要求：第一个 fulfilled 的输入决定结果；所有输入 rejected 时按输入顺序通过 `AggregateError` 返回原因；支持普通值和空可迭代对象。说明时间复杂度与边界条件。

## 4. React Server Components 的安全边界

- 题型：场景设计 / 框架原理
- 难度：高级
- 来源或考察方向：[React Server Components](https://react.dev/reference/rsc/server-components)
- 题目：某 Next.js 项目把数据库查询放进 Server Component，并通过 props 向 Client Component 传数据。请说明哪些代码和数据不会进入浏览器、哪些数据仍可能泄漏，以及如何设计 DTO、鉴权、缓存键和日志，避免密钥、越权数据或跨用户缓存污染。

## 5. Vue 3 响应式解构与 `toRef` / `toRefs`

- 题型：八股文 / 排错题
- 难度：中级
- 来源或考察方向：[Vue Reactivity Utilities](https://vuejs.org/api/reactivity-utilities.html)
- 题目：为什么直接解构 `reactive` 对象的基本类型属性可能丢失响应性，而解构对象属性后修改其内部字段有时仍能触发更新？比较 `toRef`、`toRefs`、`computed` 的语义，并给出组合式函数返回状态时的选择原则。

## 6. View Transition API 与无障碍降级

- 题型：浏览器 / 场景题
- 难度：中高级
- 来源或考察方向：[MDN View Transition API](https://developer.mozilla.org/docs/Web/API/View_Transition_API)
- 题目：设计一个列表到详情页的共享元素转场。说明同文档与跨文档 View Transition 的基本流程、快照阶段对 DOM 更新的要求、如何处理并发导航，以及在不支持 API 或用户开启 `prefers-reduced-motion` 时如何降级。

## 7. WebAuthn / Passkey 登录流程与威胁模型

- 题型：网络安全 / 场景设计
- 难度：高级
- 来源或考察方向：[WebAuthn Level 3](https://www.w3.org/TR/webauthn-3/)
- 题目：从前端、服务端和认证器三方描述 Passkey 的注册与登录流程。解释 challenge、RP ID、origin、用户验证、签名计数器各自防御什么风险，并说明为什么前端不能自行判定认证成功，以及如何避免账户枚举和重放攻击。

## 8. 用有限状态机治理复杂表单

- 题型：项目经验 / 场景设计
- 难度：中高级
- 来源或考察方向：综合整理
- 题目：一个多步骤开户表单包含草稿保存、异步校验、文件上传、人工审核、失败重试和返回修改。请用有限状态机描述核心状态与事件，说明如何阻止非法状态组合、处理异步竞态、支持刷新恢复，并比较状态机与多个布尔变量方案的取舍。

## 9. 算法：每日温度与单调栈

- 题型：算法题
- 难度：中级
- 来源或考察方向：[LeetCode 739 Daily Temperatures](https://leetcode.com/problems/daily-temperatures/)
- 题目：给定每天温度数组 `temperatures`，返回数组 `answer`，其中 `answer[i]` 表示第 `i` 天之后经过多少天会出现更高温度，不存在则为 `0`。要求实现 `O(n)` 解法，解释栈中保存什么以及为何每个元素最多进出栈一次。

## 10. Next.js 按标签失效与并发更新一致性

- 题型：框架原理 / 场景设计
- 难度：高级
- 来源或考察方向：[Next.js Cache Components](https://nextjs.org/docs/app/getting-started/cache-components)
- 题目：商品详情、分类列表和用户购物车分别有不同缓存生命周期。请设计缓存标签与失效策略，比较 `revalidateTag`、`updateTag`、`revalidatePath` 的适用场景，并分析并发写入、读己之写和过度失效问题。

## 11. Node.js 线上内存泄漏定位

- 题型：项目经验 / 故障排查
- 难度：高级
- 来源或考察方向：[Node.js Diagnostics](https://nodejs.org/en/learn/diagnostics/memory)
- 题目：Node.js BFF 的 RSS 持续上涨并最终被容器 OOMKilled，但 `heapUsed` 增长不明显。请给出从监控、复现、Heap Snapshot、Allocation Profile、原生内存和事件监听器等方向展开的定位路径，并说明采集快照的生产风险。

## 13. GraphQL N+1 与请求级 DataLoader

## 14. RAG 流式回答中的引用一致性

## 15. Agent 的中断、恢复与副作用幂等

- 题型：Agent 开发 / 场景设计
- 难度：高级
- 来源或考察方向：综合整理
- 题目：一个 Agent 会查库存、创建订单并发送通知，执行到创建订单后进程崩溃。请设计可恢复执行模型，说明 checkpoint 应保存什么、如何区分已完成与待执行工具调用、如何使用幂等键和副作用日志避免重复下单，以及恢复前为何需要重新校验权限和外部状态。
