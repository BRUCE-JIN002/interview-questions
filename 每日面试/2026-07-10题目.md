# 2026-07-10 每日面试题

> 分布：前端核心 9 题、全栈工程 4 题、AI / Agent 2 题。

## 1. `Promise.try` 的同步调用与错误归一

- 题型：JavaScript / Promise 八股题
- 难度：中级
- 来源或考察方向：[MDN Promise.try](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise/try)
- 题目正文：`Promise.try(fn)`、`Promise.resolve().then(fn)`、`new Promise(resolve => resolve(fn()))` 在回调执行时机、同步异常捕获、返回 thenable 处理上有什么差异？如果要兼容旧浏览器，你会如何实现一个近似版本，并说明它不能完全等价的地方。

## 2. TypeScript 可辨识联合与穷尽性检查

- 题型：TypeScript 类型建模题
- 难度：中级
- 来源或考察方向：[TypeScript Narrowing](https://www.typescriptlang.org/docs/handbook/2/narrowing.html#discriminated-unions)
- 题目正文：一个订单状态包含 `created`、`paid`、`shipped`、`cancelled`，每种状态携带不同字段。请用 TypeScript 建模，并实现 `renderOrderStatus(order)`，要求新增状态时编译期能提示遗漏分支。为什么在这种场景下可辨识联合通常优于大量可选字段？

## 3. `Intl.Segmenter` 与前端文本截断

- 题型：浏览器 API / 国际化场景题
- 难度：中级
- 来源或考察方向：[MDN Intl.Segmenter](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Intl/Segmenter)
- 题目正文：聊天列表需要按“用户感知字符”截断昵称，昵称可能包含中文、英文、Emoji、组合音标和旗帜。为什么不能直接用 `str.length` 或 `slice(0, n)`？如何利用 `Intl.Segmenter` 或降级方案实现更合理的截断，并说明性能和兼容性取舍。

## 4. CSS Anchor Positioning 的弹层定位边界

- 难度：中高级
- 来源或考察方向：[MDN CSS anchor positioning](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_anchor_positioning)

## 5. React `cacheSignal` 与服务端数据读取取消

## 7. Web Worker、Transferable 与大数据渲染

## 9. 手写 LFU Cache

- 题型：算法 / 数据结构题
- 难度：高级
- 来源或考察方向：综合整理；缓存淘汰策略、哈希表与链表
- 题目正文：实现 `LFUCache(capacity)`，支持 `get(key)` 和 `put(key, value)`，要求平均 O(1)。当访问频次相同时淘汰最久未使用的 key。请说明需要维护哪些映射结构，以及 `capacity = 0`、更新已有 key、频次桶为空时的边界。
