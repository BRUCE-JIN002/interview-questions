# 2026-07-06 全栈偏前端面试题参考答案

## 1. JavaScript Import Attributes 与 JSON Module

### 参考答案

Import Attributes 明确声明模块类型，宿主会校验响应 MIME，避免本想加载数据却把响应当脚本执行。JSON Module 属于模块图：静态导入参与依赖分析，动态导入也会按模块身份缓存，通常只提供默认导出。`fetch()` 是运行时请求，可自行控制 headers、取消、重试和 HTTP 缓存策略，适合会变化的数据；JSON Module 更适合随代码部署的静态配置。

### 关键考点

- 模块类型与 MIME 校验；模块缓存不是业务数据刷新机制。
- 静态导入发生在模块求值前，`fetch()` 返回 Promise。

### 常见追问或变体

- 动态 `import()` 如何传 `with`？构建工具和 Node.js 版本兼容性如何处理？

## 2. TypeScript 精确可选属性

### 参考答案

开启该选项后，`foo?: string` 表示属性可以不存在，但若存在必须是 `string`；不能显式写 `foo: undefined`。`foo: string | undefined` 则要求属性存在，只是值可为 `undefined`。两者在 `"foo" in obj`、`Object.keys`、对象展开和 `Object.assign` 中运行时行为不同。PATCH DTO 应明确“未传”与“清空”的协议，例如用 `null` 表示清空。

### 关键考点

- 可选性描述属性是否存在，联合类型描述值域。
- 该选项需要 `strictNullChecks`，但不属于 `strict` 默认集合。

### 常见追问或变体

- `Partial<T>`、表单默认值和 JSON 序列化会受到什么影响？

## 3. 手写支持 TTL 的 memoize

### 参考答案

```ts
function memoize<A extends unknown[], R>(fn: (...args: A) => R, ttl: number) {
  const cache = new Map<string, { value: R; expires: number }>();
  return (...args: A): R => {
    const key = JSON.stringify(args);
    const hit = cache.get(key);
    const now = Date.now();
    if (hit && hit.expires > now) return hit.value;

    const value = fn(...args);
    cache.set(key, { value, expires: now + ttl });
    if (value instanceof Promise) {
      value.catch(() => {
        if (cache.get(key)?.value === value) cache.delete(key);
      });
    }
    return value;
  };
}
```

同一个 Promise 立即入缓存可合并并发请求；失败时只删除仍指向该 Promise 的条目，避免误删后续值。生产实现不能默认用 JSON 字符串化所有参数，应按业务键函数或对象身份建多级 `Map/WeakMap`，并定期淘汰过期项。

### 关键考点

- 并发复用、失败驱逐、TTL 和键稳定性。

### 常见追问或变体

- 如何加入最大容量、LRU、主动清理与 `clear(key)`？

## 4. 浏览器跨源隔离与 SharedArrayBuffer

### 参考答案

`SharedArrayBuffer` 可建立高精度共享内存通道，受 Spectre 类攻击影响，因此浏览器通常要求隔离环境。COOP 将顶层页面放入独立浏览上下文组，切断不安全的 opener 关系；COEP 要求嵌入的跨源资源通过 CORS 或 CORP 明确授权。两者满足后用 `crossOriginIsolated` 做能力检测。第三方资源缺少 CORS/CORP、弹窗依赖 `window.opener`、CDN 响应头不一致都会破坏接入。

### 关键考点

- COOP 隔离窗口关系，COEP 约束子资源；必须检查真实响应头。

### 常见追问或变体

- `COEP: credentialless` 与 `require-corp` 有什么区别？

## 5. React Effect 中的请求竞态

### 参考答案

Effect 每次启动请求，并在清理函数中标记当前请求失效或调用 `controller.abort()`；回写前确认请求仍有效。忽略旧响应能保证 UI 正确但不节省网络；AbortController 同时节省客户端资源，但服务端未必停止工作；成熟请求库还能提供缓存、去重、失效和 Suspense/SSR 集成。错误处理中应排除 `AbortError`。

```tsx
useEffect(() => {
  const c = new AbortController();
  fetch(`/api/search?q=${encodeURIComponent(q)}`, { signal: c.signal })
    .then(r => r.json())
    .then(setData)
    .catch(e => e.name !== "AbortError" && setError(e));
  return () => c.abort();
}, [q]);
```

### 关键考点

- Effect 清理先于下一次同类 Effect 执行，也在卸载时执行。

### 常见追问或变体

- Strict Mode 开发环境双执行暴露了什么问题？

## 6. Vue 3 `watch` 的清理与竞态控制

### 参考答案

```ts
watch(query, async (value, _old, onCleanup) => {
  const c = new AbortController();
  onCleanup(() => c.abort());
  try {
    result.value = await fetch(`/api?q=${encodeURIComponent(value)}`, {
      signal: c.signal
    }).then(r => r.json());
  } catch (e: any) {
    if (e.name !== "AbortError") error.value = e;
  }
});
```

依赖再次变化或 watcher 停止前会调用清理逻辑。默认 `pre` 适合多数状态同步；需要读取组件更新后的 DOM 用 `post`；`sync` 无批处理，适合极少量必须同步的状态，不应用于高频数组变化。

### 关键考点

- 清理必须同步注册，不能在 `await` 后才注册。

## 7. HTTP/3 与 QUIC 的收益和迁移边界

### 参考答案

HTTP/2 多条流共享一个 TCP 字节流，单个丢包会阻塞同连接后续数据；QUIC 在 UDP 上实现独立流，通常只阻塞受影响的流。QUIC 集成 TLS 1.3，减少握手往返；连接由 Connection ID 标识，网络地址变化时可迁移。UDP 可能被企业网络或中间设备阻断，因此需通过 Alt-Svc/HTTPS 记录尝试 HTTP/3 并保留 HTTP/2 回退。传输层改进不等于业务操作可安全重放，非幂等请求仍需幂等键。

### 关键考点

- 消除的是跨流传输层队头阻塞，不是所有排队问题。

### 常见追问或变体

- 0-RTT 有什么重放风险？

## 8. 前端离线写入与冲突合并

### 参考答案

IndexedDB 保存实体快照和 append-only 操作日志，每个操作带 clientId、operationId、baseVersion、时间和临时 ID。恢复网络后按依赖顺序批量同步，服务端用 operationId 幂等，并返回临时 ID 到真实 ID 的映射。版本不一致时，简单字段可按字段版本或明确规则合并；删除与编辑等语义冲突进入人工处理，不能静默“最后写入覆盖”。UI 将本地状态与同步状态分离，展示 pending、conflict、failed，并允许重试或撤销。

### 关键考点

- 离线优先不是只缓存响应，核心是可重放操作与冲突语义。

### 常见追问或变体

- 多设备同时编辑时，何时需要 CRDT？

## 9. 算法：二叉树最近公共祖先

### 参考答案

```ts
function lowestCommonAncestor(
  root: TreeNode | null,
  p: TreeNode,
  q: TreeNode
): TreeNode | null {
  if (!root || root === p || root === q) return root;
  const left = lowestCommonAncestor(root.left, p, q);
  const right = lowestCommonAncestor(root.right, p, q);
  if (left && right) return root;
  return left ?? right;
}
```

若两个目标分居左右子树，当前节点就是最近公共祖先；否则结果沿非空侧上传。时间 `O(n)`，递归栈 `O(h)`。若节点可能不存在，需要同时返回找到目标的数量，仅在计数为 2 时接受候选祖先。

### 关键考点

- 后序遍历、返回值语义、普通二叉树与二叉搜索树解法不同。

### 常见追问或变体

- 多次查询如何通过父指针、Euler Tour 或倍增预处理优化？

## 10. Next.js Middleware、Proxy 与鉴权边界

### 参考答案

入口拦截适合廉价的 cookie 存在性检查和重定向，但它可能被绕过，且无法表达资源级权限。页面/Server Component 在读取敏感数据前校验会话；Server Action 和 Route Handler 各自视为公开入口，重新验证身份、权限和输入；数据访问层集中执行 `requireUser`、租户过滤和对象级授权。客户端隐藏按钮只是体验优化，不是安全措施。

### 关键考点

- 每个写入口都重新授权；认证、角色权限、资源所有权分层。

### 常见追问或变体

- 如何避免缓存把 A 用户数据泄漏给 B 用户？

## 11. Node.js `AsyncResource` 与异步上下文丢失

### 参考答案

原生异步资源会传播 async hooks 上下文，但自行保存回调并在无关联的调度点执行时，创建和执行链可能断开。创建任务时用 `AsyncResource` 捕获上下文，执行时通过 `runInAsyncScope` 调用回调，完成后调用 `emitDestroy`。验证时并发发出大量带不同 requestId 的请求，在定时器、Promise、队列回调和错误路径记录 ID，断言每条日志只属于其请求。

### 关键考点

- 问题是异步资源边界，不应靠全局变量传 requestId。

### 常见追问或变体

- 何时优先使用 `AsyncLocalStorage.snapshot()` 或 `bind()`？

## 12. NestJS 动态模块的设计边界

### 参考答案

定义稳定的 `SDK_OPTIONS` Token；`forRoot` 注册 `useValue`，`forRootAsync` 接受 `useFactory`、`inject` 并注册异步配置。内部 Provider 根据配置创建 SDK client，只导出面向业务的适配器服务，不直接导出第三方实例。仅真正基础且全应用唯一的设施考虑 `global: true`，否则显式导入更清晰。测试时覆盖 Token 或适配器即可。

### 关键考点

- 配置、实例构造、业务适配三层分离；避免字符串 Token 冲突。

### 常见追问或变体

- 多租户需要多个不同配置的实例时怎么设计？

## 13. API 限流与分布式配额

### 参考答案

固定窗口简单但边界可双倍突发；滑动窗口更准确但存储/计算更高；令牌桶允许可控突发，适合 API。Redis 中用 Lua 脚本原子完成补充、扣减和过期，键包含租户/用户/接口；匿名流量再结合可信代理解析后的 IP。响应 `429` 和合理 `Retry-After`，客户端仅对幂等操作做带抖动退避。Redis 故障时按接口风险选择本地粗限流、fail-open 或 fail-closed，并监控拒绝率。

### 关键考点

- 限流维度、原子性、可信代理和故障策略。

### 常见追问或变体

- 热点大客户如何避免单键瓶颈？

## 14. RAG 检索结果的前端证据面板

### 参考答案

消息正文使用稳定 messageId；引用用 citationId 单独建模，包含 documentId、chunkId、版本、范围和状态。流事件分为 text delta、citation upsert、citation revoke、done/error，前端按事件序号幂等归并。引用未验证前显示占位，不把模型生成的 URL 直接当可信链接；服务端根据检索结果签发可访问引用，跳转时再次检查权限和文档版本。被撤销引用保留审计状态而非静默换成另一文档。

### 关键考点

- 文本流和证据流解耦；引用身份必须来自受信系统。

### 常见追问或变体

- 如何衡量“回答有引用但引用不支持结论”？

## 15. MCP Roots 与 Agent 文件访问边界

### 参考答案

客户端通过 Roots 告知服务端当前允许或相关的目录，服务端据此缩小工具可见范围；但服务端必须把它当提示而非授权证明。每次访问先解析为规范绝对路径，拒绝 `..` 越界，并在打开目标时防止符号链接或 TOCTOU 逃逸；写入、覆盖、删除等高风险动作逐次展示最终路径并确认。工具进程还应使用 OS 沙箱和最小文件权限。

Roots 可动态变化，服务端应处理列表更新；客户端声明的根也可能恶意或配置错误，因此真正的边界必须由服务端校验、操作系统权限和用户审批共同执行。

### 关键考点

- 能力发现不等于强制授权；校验最终解析目标而非原始字符串。

### 常见追问或变体

- 如何安全处理根目录内指向根目录外的符号链接？
