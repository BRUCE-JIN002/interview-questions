## 1. Promise 与事件循环输出分析

- 题型：八股 + 代码输出
- 难度：中级
- 可能来源或考察方向：综合整理；Promise、microtask、macrotask、async/await
- 题目正文：

请分析下面代码的输出顺序，并解释每一步进入同步任务、微任务或宏任务的原因。

```js
console.log('A');

setTimeout(() => {
  console.log('B');
  Promise.resolve().then(() => console.log('C'));
}, 0);

Promise.resolve()
  .then(() => {
    console.log('D');
    return Promise.resolve('E');
  })
  .then((value) => {
    console.log(value);
    setTimeout(() => console.log('F'), 0);
  });

async function run() {
  console.log('G');
  await null;
  console.log('H');
}

run();
console.log('I');
```

## 2. 手写 Promise.allSettled

- 题型：手写实现
- 难度：中级
- 可能来源或考察方向：综合整理；Promise 并发控制、结果顺序、异常隔离
- 题目正文：

请手写 `Promise.allSettled(iterable)`。要求：

1. 保持返回结果顺序与输入顺序一致。
2. 每个结果形如 `{ status: 'fulfilled', value }` 或 `{ status: 'rejected', reason }`。
3. 空数组应立即 resolve 为 `[]`。
4. 输入项可以是普通值、thenable 或 Promise。

## 3. TypeScript 类型体操：DeepReadonly

- 题型：类型编程
- 难度：中高级
- 可能来源或考察方向：综合整理；条件类型、映射类型、函数和数组边界
- 题目正文：

实现一个 `DeepReadonly<T>` 类型，使对象的所有嵌套属性都变为只读。要求：

1. 函数类型保持原样。
2. 数组和元组中的元素也应递归只读。
3. 基础类型不变。

示例：

```ts
type User = {
  id: number;
  profile: {
    tags: string[];
    update: () => void;
  };
};

type ReadonlyUser = DeepReadonly<User>;
```
