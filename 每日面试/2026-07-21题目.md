# 2026-07-21 全栈偏前端每日面试题

## 1. JavaScript：`Array.fromAsync` 与异步迭代消费

- 题型：原理 + 场景
- 难度：中等
- 可能来源或考察方向：[MDN Array.fromAsync](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Array/fromAsync)
- 题目：`Array.fromAsync()` 与 `Promise.all()`、`Array.from()` 分别适合什么输入？消费一个可能无限的异步迭代器时有哪些内存和取消风险？

## 2. TypeScript：断言函数与边界数据收窄

- 题型：手写 + 原理
- 难度：中等
- 可能来源或考察方向：[TypeScript Handbook: Assertion Functions](https://www.typescriptlang.org/docs/handbook/2/narrowing.html#assertion-functions)
- 题目：为来自接口的 `unknown` 值编写 `assertUser(value): asserts value is User`。它与返回 `value is User` 的类型守卫有什么控制流差异？什么时候不能只依赖 TypeScript 类型？

## 3. ES6+：`Symbol` 作为协议键的设计边界

- 题型：原理 + 设计
- 难度：中等
- 可能来源或考察方向：[MDN Symbol](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Symbol)
- 题目：组件库希望允许外部对象提供“可序列化”和“可比较”能力。为什么可用 `Symbol` 键而非字符串方法名？`Symbol.for()`、私有字段和 WeakMap 各自的隔离程度如何？

## 4. Promise：异步任务的“已开始”与“受控并发”

- 题型：原理 + 场景
- 难度：中等
- 可能来源或考察方向：综合整理
- 题目：为什么把一组已经创建的 `Promise` 交给并发池不能真正限制并发？将任务表示为 `() => Promise<T>` 后，如何同时处理同步抛错、失败策略和取消？

## 5. 浏览器：`ResizeObserver` 循环与布局抖动

- 题型：场景排障
- 难度：中等
- 可能来源或考察方向：[MDN ResizeObserver](https://developer.mozilla.org/docs/Web/API/ResizeObserver)
- 题目：线上偶发 `ResizeObserver loop completed with undelivered notifications`，同时卡片高度来回跳动。解释典型因果链，并给出不掩盖问题的定位、修复和降级方案。

## 6. 网络：`fetch` 请求体流、重定向与重试

- 题型：原理 + 场景
- 难度：较难
- 可能来源或考察方向：[MDN Using Fetch](https://developer.mozilla.org/docs/Web/API/Fetch_API/Using_Fetch)
- 题目：BFF 代理一个包含 `ReadableStream` 请求体的上传请求，为什么不能随意重试或转发两次？如何设计可重试上传协议，并避免 307/308 重定向后凭证或大请求体带来的问题？

## 7. React：`useSyncExternalStore` 的 hydration 契约

- 题型：原理 + 场景
- 难度：较难
- 可能来源或考察方向：[React useSyncExternalStore](https://react.dev/reference/react/useSyncExternalStore)

## 9. 算法：分割等和子集

- 题型：算法题
- 难度：中等
