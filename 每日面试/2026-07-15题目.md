# 2026-07-15 全栈偏前端每日面试题

## 1. JavaScript：`Atomics.waitAsync` 与共享内存异步协作

- 题型：语言特性 + 并发场景题
- 难度：高级
- 考察方向：JavaScript、SharedArrayBuffer、Web Worker
- 来源：[MDN Atomics.waitAsync](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Atomics/waitAsync)
- 题目：解释 `Atomics.waitAsync` 与 `Atomics.wait` 的区别。设计一个主线程与 Worker 通过 `SharedArrayBuffer` 协作的任务状态通知机制：如何避免主线程阻塞、处理通知早于等待的竞态，并定义超时与取消的行为？

## 2. TypeScript：显式方差标注与回调 API 设计

- 题型：类型原理 + API 设计题
- 难度：高级
- 考察方向：TypeScript、泛型、函数类型
- 来源：[TypeScript Handbook - Generics](https://www.typescriptlang.org/docs/handbook/2/generics.html)
- 题目：一个库暴露 `Producer<T>`（只产出 T）和 `Consumer<T>`（只消费 T）。解释协变、逆变与不变；何时可使用 `out` / `in` 方差标注？为什么在普通结构化类型上不能把方差标注当作“强制安全开关”？

## 3. ES6+：异步生成器的背压与提前关闭

- 题型：原理 + 手写题
- 难度：中高级
- 考察方向：AsyncIterator、Stream、资源释放
- 来源：[MDN AsyncGenerator](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/AsyncGenerator)
- 题目：分页接口需要逐页拉取数据并允许消费者随时停止。请用 async generator 设计 `pages(fetchPage)`，说明 `for await...of` 如何形成背压；消费者 `break`、请求失败或外部取消时，如何停止后续请求并清理资源？

## 4. 浏览器：长任务切片与 INP 优化

- 题型：性能排障题
- 难度：中高级
- 考察方向：事件循环、Performance API、INP
- 来源：[web.dev Optimize INP](https://web.dev/articles/optimize-inp)
- 题目：筛选面板点击后要同步处理 5 万条本地数据，交互延迟明显升高。如何用 Performance 面板和 Long Tasks 定位瓶颈？比较 `setTimeout`、`requestIdleCallback`、`scheduler.yield()` 与 Worker 的适用性，并说明为何不能只把计算拆成更多 Promise 微任务。

## 5. 网络：HTTP 条件请求与并发编辑冲突

- 题型：网络原理 + 接口设计题
- 难度：中高级
- 考察方向：HTTP、REST、并发控制
- 来源：[RFC 9110 Conditional Requests](https://www.rfc-editor.org/rfc/rfc9110.html#name-conditional-requests)
- 题目：用户在两个标签页编辑同一篇文章。如何用 `ETag`、`If-Match` 与 `412 Precondition Failed` 实现乐观并发控制？前端收到冲突后如何展示差异和重试？为什么不能仅依赖客户端的修改时间？

## 6. React：View Transition 与并发导航的状态边界

- 题型：框架原理 + 场景题
- 难度：中高级
- 考察方向：React、并发渲染、动画、可访问性
- 来源：[React ViewTransition](https://react.dev/reference/react/ViewTransition)
- 题目：商品列表点击卡片进入详情页时，希望共享元素平滑过渡且不阻塞输入。React 的 `<ViewTransition>`（或浏览器 View Transition API）应如何与 `startTransition` 配合？哪些状态不应被动画“冻结”，以及如何为 `prefers-reduced-motion` 提供降级？

## 7. Vue 3：`defineExpose` 的组件边界与命令式能力

- 题型：框架设计题
- 难度：中级
- 考察方向：Vue 3、组件封装、TypeScript
- 来源：[Vue defineExpose](https://vuejs.org/api/sfc-script-setup.html#defineexpose)
- 题目：封装一个上传组件，父组件偶尔需要调用 `openFileDialog()`、`abort()` 和 `reset()`。在 `<script setup>` 中如何通过 `defineExpose` 暴露最小命令式接口？哪些内部状态不应暴露，为什么不能用父组件直接改子组件响应式状态来“图省事”？

## 8. 手写：支持订阅取消的 Observable

- 题型：手写实现
- 难度：中高级
- 考察方向：JavaScript、异步流、资源清理
- 来源：综合整理
- 题目：实现最小版 `Observable`：`subscribe(observer)` 返回 `unsubscribe()`；生产者可同步或异步发送 `next`、`error`、`complete`。要求取消后不再通知，`error` 或 `complete` 后自动关闭，并确保清理函数至多执行一次。

## 9. 算法：最短编辑操作序列

- 题型：算法题
- 难度：中高级
- 考察方向：动态规划、字符串
- 来源：综合整理
- 题目：给定 `source` 和 `target`，除求最小编辑距离外，还要返回一组具体操作（插入、删除、替换）。说明 DP 状态、转移、如何回溯操作序列，以及在只需要距离时怎样压缩空间。

## 10. Next.js：`generateMetadata`、数据去重与 SEO 一致性

- 题型：全栈工程题
- 难度：中高级
- 考察方向：Next.js、Server Components、SEO、缓存
- 来源：[Next.js generateMetadata](https://nextjs.org/docs/app/api-reference/functions/generate-metadata)
- 题目：商品详情页的 `generateMetadata` 与页面组件都需要获取商品数据。如何避免一次请求渲染时的重复访问，同时确保标题、canonical、Open Graph 与页面正文来自同一版本？当商品下架或数据失败时，如何选择 `notFound`、降级 metadata 与缓存策略？

## 11. Node.js：ESM Loader 的能力边界与生产风险

- 题型：运行时原理 + 工程决策题
- 难度：高级
- 考察方向：Node.js、ESM、模块加载、安全
- 来源：[Node.js Modules: Customization hooks](https://nodejs.org/api/module.html#customization-hooks)
- 题目：团队想用 Node ESM loader 在运行时解析路径别名、加载远程配置或做源码转换。loader hooks 在模块解析/加载链路中的位置是什么？它会带来哪些启动性能、缓存、一致性和供应链风险？哪些能力更应该在构建期或启动期完成？

## 12. NestJS：Outbox 模式与领域事件可靠投递

- 题型：后端架构题
- 难度：高级

- 来源：综合整理
- 题目：订单创建后需要发优惠券、积分和通知事件，直接在事务内发 MQ 可能丢消息或重复。请设计 NestJS 中的 transactional outbox：表结构核心字段、写入与轮询/CDC 发布流程、消费者去重，以及失败告警和回放策略。

## 13. 数据库：读写分离下的“读己之写”一致性
