# 2026-07-22 全栈偏前端每日面试题

## 1. JavaScript：Proxy 的不变量与响应式边界

- 题型：原理 + 场景
- 难度：较难
- 可能来源或考察方向：[MDN Proxy](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Proxy)
- 题目：为什么 `Proxy` 的 `get`、`ownKeys`、`defineProperty` 等 trap 不能任意返回结果？列举一个违反对象不变量会抛 `TypeError` 的例子。用 Proxy 写响应式系统时，哪些对象不应深度代理？

## 2. TypeScript：`satisfies` 与配置对象的类型约束

- 题型：原理 + 手写
- 难度：中等
- 可能来源或考察方向：[TypeScript 4.9 satisfies](https://www.typescriptlang.org/docs/handbook/release-notes/typescript-4-9.html)
- 题目：比较类型标注、`as` 断言与 `satisfies`。为路由配置既检查字段拼写，又保留每个路由字面量的精确类型，应如何写？

## 3. ES6+：`structuredClone`、转移与不可克隆值

- 题型：原理 + 场景
- 难度：中等
- 可能来源或考察方向：[MDN structuredClone](https://developer.mozilla.org/docs/Web/API/Window/structuredClone)
- 题目：`structuredClone` 与 JSON 深拷贝的能力和限制分别是什么？将 `ArrayBuffer` 传给 Web Worker 时，clone 和 transfer 有什么性能与所有权差异？

## 4. 事件循环：渲染、微任务与长任务切片

- 题型：原理 + 性能优化
- 难度：较难
- 可能来源或考察方向：综合整理
- 题目：点击后同步计算 300ms，页面为何不能及时展示 loading？说明 task、microtask、渲染机会的关系，并设计一个既可取消又不阻塞交互的分片计算方案。

## 5. 浏览器：跨标签页状态同步与一致性

- 题型：场景设计
- 难度：中等
- 可能来源或考察方向：[MDN BroadcastChannel](https://developer.mozilla.org/docs/Web/API/BroadcastChannel)
- 题目：用户在两个标签页同时编辑购物车和登录状态。如何选择 `BroadcastChannel`、`storage` 事件、SharedWorker 或服务端推送？怎样处理消息丢失、乱序和登出后的敏感缓存？

## 6. 网络：CORS 预检缓存与凭证请求

- 题型：原理 + 排障
- 难度：中等
- 可能来源或考察方向：[MDN CORS](https://developer.mozilla.org/docs/Web/HTTP/Guides/CORS)
- 题目：前端带 cookie 调用 BFF 时，为什么服务端不能把 `Access-Control-Allow-Origin` 设为 `*`？预检请求何时产生、如何安全利用 `Access-Control-Max-Age`，又为何仍不能把 CORS 当作鉴权？

## 7. React：并发渲染下的 Transition 与请求竞争

- 题型：场景设计
- 难度：较难
- 可能来源或考察方向：[React useTransition](https://react.dev/reference/react/useTransition)
- 题目：搜索框输入触发列表请求，慢请求偶尔覆盖快请求结果。`useTransition` 能解决哪些体验问题、不能解决哪些数据一致性问题？给出含 `AbortController` 或请求序号的处理方案。

## 8. Vue 3：`defineModel` 与受控组件契约

- 题型：工程实践
- 难度：中等
- 可能来源或考察方向：[Vue Component v-model](https://vuejs.org/guide/components/v-model.html)
- 题目：封装支持 `v-model:open`、修饰符和异步关闭确认的弹窗。`defineModel` 如何对应 prop/event？为什么组件不能直接修改 prop，异步确认期间怎样避免父子状态回弹？

## 9. 算法：LRU 缓存

- 题型：手写算法
- 难度：中等
- 可能来源或考察方向：[LeetCode 146](https://leetcode.com/problems/lru-cache/)
- 题目：手写固定容量的 LRU Cache，要求 `get`、`put` 均为 O(1)。解释哈希表与双向链表如何协作，并说明更新已有 key、容量为 0 时的边界。

## 10. Next.js：Server Actions 的授权与重放防护

- 题型：框架安全 + 场景
- 难度：较难
- 可能来源或考察方向：[Next.js Server Actions](https://nextjs.org/docs/app/getting-started/updating-data)
- 题目：为什么 Server Action 不能因为“函数在服务端”就省略鉴权和输入校验？设计一个更新资料的 Action，说明身份、资源归属、CSRF/重放、错误返回和缓存失效分别放在哪里。

## 11. Node.js：背压与流式代理
