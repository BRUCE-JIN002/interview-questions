# 2026-07-16 全栈偏前端面试题参考答案

## 1. `Array.prototype.sort` 的稳定性与比较器契约

- 参考答案：ES2019 起规范要求稳定排序；相等元素保持原有相对顺序。比较器应对同一对输入给出稳定结果，满足反自反、反对称与传递性；返回负数、零、正数分别表示前、等、后。
- 关键考点：不要用 `a > b ? 1 : -1` 处理相等值；不传比较器会按 UTF-16 字符串排序。
- 常见追问：不合法比较器会让不同引擎或同一引擎不同数据规模得到不一致顺序，分页、虚拟列表 key 和缓存命中都会异常。

## 2. `unknown`、`any` 与边界数据解析

- 参考答案：`any` 会关闭后续类型检查；`unknown` 强制先收窄。外部输入都可能与声明不符，解析层应把 `unknown` 校验为领域类型，再交给业务层。
- 关键考点：TypeScript 类型在运行时不存在；需要类型守卫或 schema 校验。
- 代码实现：

```ts
type User = { id: string; name: string };

function parseUser(value: unknown): User {
  if (!value || typeof value !== 'object') throw new Error('invalid user');
  const { id, name } = value as Record<string, unknown>;
  if (typeof id !== 'string' || typeof name !== 'string') throw new Error('invalid user');
  return { id, name };
}
```

## 3. `Map` 的 SameValueZero 键比较与对象键陷阱

- 参考答案：`Map` 使用 SameValueZero：`NaN` 等于自身，`-0` 与 `0` 相同；对象按引用身份比较，两个字面量即使结构一致也是两个键。
- 关键考点：适合任意类型键、频繁增删、需要可靠 size 或避免原型链键冲突的场景；普通对象适合固定的字符串属性结构。
- 常见追问：结构相等键要用稳定序列化键或额外的规范化/哈希层，不能期待 `Map` 深比较。

## 4. 合成层、`transform` 动画与显存风险

- 参考答案：`transform`、`opacity` 常可跳过布局和绘制，仅在合成阶段更新；但元素是否独立成层由浏览器决定，并非绝对免费。
- 关键考点：大面积或过多图层会增加纹理上传和 GPU 内存，滚动时可能触发栅格化抖动；不要滥用 `will-change`。
- 常见追问：用 Performance 看帧和 Long Task，用 Rendering 的 Paint Flashing / Layers 检查是否产生 paint 与图层数量，在目标低端机实测。

## 5. HTTPS 页面中的 Mixed Content 与第三方资源治理

- 参考答案：可执行的主动混合内容（如脚本）通常被阻止；部分被动内容可能被升级或阻止，不能依赖浏览器兼容策略。第三方资源一律用 HTTPS、CSP 限制域名，关键能力准备自有兜底。
- 关键考点：构建期扫描 HTTP URL；CSP `upgrade-insecure-requests` 是辅助，不替代资源治理；SRI 适用于内容固定的第三方静态资源。
- 常见追问：运行时记录资源加载错误和 CSP report，图片降级占位、第三方 iframe 提示不可用，不能把 token 放到第三方 URL。

## 6. React 受控输入在并发更新下的输入体验

- 参考答案：输入 state 必须同步更新，昂贵列表使用 `useDeferredValue` 或 `startTransition` 延后；请求使用 `AbortController` 或递增 request id，只有当前请求可提交结果。
- 关键考点：不要把受控输入的 `setValue` 放进 transition；服务端结果需要和当前 query 绑定。
- 常见追问：本地筛选可 memo 化、分片或虚拟化；网络搜索增加 debounce 但不能代替竞态处理。

## 7. `KeepAlive` 缓存生命周期与资源释放

- 参考答案：首次进入 `mounted` 后 `activated`；切走触发 `deactivated` 而非卸载；缓存淘汰才触发 `unmounted`。在 `activated` 恢复轮询/订阅，在 `deactivated` 停止它们；草稿保存到 store 或持久层并设置过期策略。
- 关键考点：`onDeactivated` 也会在最终卸载时调用，清理逻辑应幂等。
- 常见追问：用 `include`、`exclude`、`max` 限制缓存；不能只依赖 `unmounted` 清 WebSocket。

## 8. 实现可组合的 `pipeAsync`

- 参考答案：用 `Promise.resolve` 归一同步/异步返回值，通过 `reduce` 串行传递；回调抛错会自动变为 reject。
- 代码实现：

```ts
function pipeAsync<T>(...fns: Array<(value: any) => any>) {
  return (input: T) => fns.reduce(
    (chain, fn) => chain.then(fn),
    Promise.resolve(input),
  );
}
```

- 常见追问：若要并行应改为显式组合器；若需取消，函数签名传入 `AbortSignal` 并由各步骤配合停止。

## 9. 合并 K 个升序链表

- 参考答案：小顶堆始终保存每条链表当前头节点，弹出最小节点并推入其 next，时间 `O(N log k)`，额外空间 `O(k)`；分治两两合并同为 `O(N log k)`，递归栈 `O(log k)`。
- 关键考点：空链表不能入堆；复用原节点时先保存 `next`，避免断链。
- 常见追问：当 `k` 很小可用分治；堆解法更适合流式到达的多路有序数据。

## 10. Next.js 国际化路由与缓存键设计

- 参考答案：Middleware 只做 locale 协商和规范化重定向；`[lang]` 进入路由参数。价格读取必须显式纳入 locale、region、会员等级和货币，私有等级不应进入可共享缓存。
- 关键考点：`Vary` / CDN key 要覆盖影响响应的公开维度；服务端以受信任 cookie/session 取等级，不能信任客户端传参。
- 常见追问：登录态价格可动态渲染或按用户分区；缓存失效同时区分语言与地区 tag。

## 11. Node.js 事件循环阻塞与 CPU 密集型接口拆分

- 参考答案：用 event loop delay、CPU profile、p99 与火焰图确认同步计算占据主线程。短且可控的 CPU 任务放 worker pool；需要强隔离或执行不可信二进制用子进程；可延迟的批处理进入队列。
- 关键考点：worker 数量受 CPU 核数和内存约束；传大数据优先 Transferable 或共享缓冲，避免复制抵消收益。
- 常见追问：压缩也可能阻塞；先限制输入大小和并发，给请求设置超时与降级，再决定架构拆分。

## 12. NestJS RBAC 与资源级授权的落点

- 参考答案：Guard 负责认证和粗粒度角色门禁；服务层集中表达“是否可访问该项目”的业务规则；查询必须带 `projectId`/tenant 条件，数据库权限或 RLS 可作为最后防线。
- 关键考点：不能只根据前端传入的资源 ID 判断；授权结论应可审计。
- 常见追问：把权限结果放到请求上下文可减少重复查库，但要设置短生命周期并在角色变更后失效。

## 13. Webhook 消费的签名、去重与重放

- 参考答案：在读取原始 body 后以共享密钥验证时间戳和签名；以 provider event id 做唯一键，先持久化原始事件与状态，再异步执行业务处理。消费者应幂等并允许按事件重放。
- 关键考点：限制时间窗防重放，记录验签失败原因但不记录密钥；乱序事件按业务版本号/状态机拒绝倒退。
- 常见追问：快速返回 2xx 避免对方重试风暴；死信与管理页提供可追踪状态、人工重试和审计。

## 14. 对话中的“引用即证据”交互设计

- 参考答案：回答中引用编号必须绑定可定位的 source id；点击打开原文片段、标题、时间和权限说明。流式阶段先显示“检索中/证据待加载”，证据失败时明确标识而不是保留伪引用。
- 关键考点：引用内容应可复制、可跳转、可反馈；权限变化时重新鉴权，不能把无权原文留在缓存中。
- 常见追问：提供“此结论不被引用支持”的反馈并回传 trace id、回答版本和引用 id，进入评测/修复闭环。

## 15. 生产 Agent 的离线评测与发布门禁

- 参考答案：评测集应含真实脱敏任务、边界案例、提示注入和危险写操作；分别评估任务完成度、引用/工具选择、参数正确性、拒绝率和延迟成本。模型或提示词变更按分层阈值阻断发布。
- 关键考点：写 CRM 等高风险工具必须评估“是否应调用”和“调用参数是否正确”，并要求人工复核样本；不能只看最终文本质量。
- 常见追问：灰度后按版本记录 trace、工具调用、失败类型和人工纠正率；超过风险阈值自动降级为只读或转人工。
