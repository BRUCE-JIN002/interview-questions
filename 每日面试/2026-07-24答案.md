# 2026-07-24 全栈偏前端每日面试参考答案

## 1. JavaScript：Top-level `await`、模块图与循环依赖

**参考答案：**含顶层 `await` 的模块会变成异步模块；依赖它的模块必须等待其求值完成后才能继续。循环中若某一方在初始化前读取另一方的导出，可能触发暂时性死区错误，或形成等待链。用导入图、启动日志和最小复现定位；把异步初始化移到显式 `init()`、在入口集中 await，或拆开共享依赖来消除环。

**关键考点：**模块链接和求值、live binding、异步依赖会向上游传播。

**常见追问或变体：**顶层 `await` 只能用于 ESM；不要用它隐式承载关键服务的无限重试。

## 2. TypeScript：声明合并、模块扩展与类型污染边界

**参考答案：**仅需给某包补类型时，在 `.d.ts` 中 `import 'package-name'` 后用 `declare module 'package-name'` 扩展接口；不要写成无模块上下文的全局声明。应用专属字段更适合在本地 wrapper 类型中表达，避免全局可见。扩展只改变编译期类型，拦截器或请求构造处仍要把 `traceId` 实际写入 header。

**关键考点：**模块作用域、声明合并、类型与运行时分离。

**常见追问或变体：**`declare global` 只用于真正跨应用的全局契约，并应置于模块文件中。

## 3. ES6+：`using`、`Symbol.dispose` 与异步资源释放

**参考答案：**`using` 在离开块作用域时调用资源的 `Symbol.dispose`，`await using` 会等待 `Symbol.asyncDispose`；多个资源按后声明先释放的 LIFO 顺序处理，即使代码抛错也会尝试清理。未支持时把获取和释放成对放入 `try/finally`，并明确谁拥有资源、谁负责关闭。

**关键考点：**确定性释放、异常路径、异步 cleanup。

**常见追问或变体：**释放失败也需要记录；不能依赖 GC/finalizer 来关闭连接或锁。

## 4. 浏览器性能：Long Task、INP 与 `scheduler.postTask`

**参考答案：**在 Performance trace 中查看交互到下一次绘制的关键路径，Long Task 的火焰图可区分脚本、样式和布局；同时排查第三方脚本。把纯计算移至 Worker；必须在主线程完成的工作按批切片，绘制相关的下一步放 `requestAnimationFrame`，可延后的低优先级任务用 `scheduler.postTask({ priority: 'background' })`。每片要足够短并在高优先级输入到来时让出。

**关键考点：**INP 看最差交互、任务切片不是无限队列、Worker 通信也有成本。

**常见追问或变体：**`requestIdleCallback` 不保证及时执行，不能承载用户可见的关键流程。

## 5. 网络：Range 请求、断点续传与内容完整性

**参考答案：**服务端声明 `Accept-Ranges: bytes`；客户端带 `Range: bytes=start-`，成功返回 `206` 和 `Content-Range`。恢复前保存初始 `ETag`，并用 `If-Range` 绑定版本；若版本已变，服务端应回退完整响应，客户端丢弃旧分片重下。多段 range 需正确处理 multipart 边界，CDN 的 range 行为也要压测验证。

**关键考点：**字节区间、版本校验、不可混拼不同实体。

**常见追问或变体：**鉴权下载也必须校验每次 range 请求，不能只校验第一次。

## 6. React：Strict Mode 下 Effect 的重复执行与外部系统同步

**参考答案：**开发模式的额外 setup → cleanup → setup 用于暴露不对称的副作用。Effect 应返回能关闭本次连接、取消本次订阅的 cleanup，并让连接函数可重复创建；不要用 ref 绕过检查，否则真实的挂载/卸载仍会泄漏。由用户点击触发的购买、埋点等一次性动作应放事件处理器；Effect 只同步当前 UI 状态与外部系统。

**关键考点：**Effect 生命周期、幂等 cleanup、渲染原因和用户事件分离。

**常见追问或变体：**生产通常不会做该开发检查，但修复仍必要，因为路由切换和依赖变化也会重跑 Effect。

## 7. Vue 3：错误捕获边界与异步组件的降级策略

**参考答案：**`defineAsyncComponent` 提供加载、超时和错误 UI；`onError` 可在有限次数内按错误类型重试。局部 `onErrorCaptured` 负责把子树切到降级态并可返回 `false` 阻止继续传播；全局 handler 兜底记录未处理错误。上报应带路由、构建版本、组件栈、错误类别和请求 id，不含敏感数据；错误 fallback 不应再立即加载同一失败模块。

**关键考点：**局部隔离优先、错误传播、有限重试和可观测性。

**常见追问或变体：**网络离线与代码异常的恢复策略不同，前者可给用户手动重试。

## 8. 手写：带 TTL 和 O(1) 淘汰的 LRU 缓存

**参考答案：**哈希表保存 key 到链表节点，双向链表头表示最近使用、尾表示最久未使用；`get` 命中后移动到头，过期则删除并 miss；`set` 更新或插入头，超容量删除尾。每次操作均为 O(1)。

```ts
class LRUCache<V> {
  private map = new Map<string, { key: string; value: V; expiresAt: number; prev?: any; next?: any }>();
  private head: any = {}; private tail: any = {};
  constructor(private capacity: number) { this.head.next = this.tail; this.tail.prev = this.head; }
  private unlink(n: any) { n.prev.next = n.next; n.next.prev = n.prev; }
  private touch(n: any) { this.unlink(n); n.prev = this.head; n.next = this.head.next; n.next.prev = n; this.head.next = n; }
  get(key: string): V | undefined { const n = this.map.get(key); if (!n) return; if (n.expiresAt <= Date.now()) { this.unlink(n); this.map.delete(key); return; } this.touch(n); return n.value; }
  set(key: string, value: V, ttlMs = Infinity) { let n = this.map.get(key); if (n) { n.value = value; n.expiresAt = Date.now() + ttlMs; this.touch(n); return; } n = { key, value, expiresAt: Date.now() + ttlMs }; this.map.set(key, n); n.prev = this.head; n.next = this.head.next; n.next.prev = n; this.head.next = n; if (this.map.size > this.capacity) { const old = this.tail.prev; this.unlink(old); this.map.delete(old.key); } }
}
```

**关键考点：**Map + 双链表、惰性过期、空值缓存和 singleflight 防击穿。

**常见追问或变体：**TTL 需要加抖动以防同时失效；可用定时清扫控制长期未访问的过期节点。

## 9. 算法：数据流中的第 K 大元素

**参考答案：**维护一个大小最多为 k 的最小堆：加入后若堆不足 k 就入堆；否则仅当新值大于堆顶时替换堆顶。堆顶始终是当前第 k 大，单次复杂度 O(log k)，空间 O(k)。全量排序每次至少要处理全部历史数据。

**关键考点：**第 k 大与最小堆、流式复杂度、边界 `k > n`。

**常见追问或变体：**若要删除任意历史值，可用延迟删除哈希配合两个堆，或使用支持 order statistic 的平衡树。

## 10. Next.js：客户端组件边界与第三方浏览器 SDK

**参考答案：**页面保持 Server Component，服务端读取地点和权限后传递纯 JSON 给一个小的 Client Component；该组件通过 `dynamic(..., { ssr: false })` 或 Effect 内动态 import 浏览器 SDK，挂载时建实例、cleanup 时销毁。整个 page 设为客户端会把更多依赖送入浏览器，并失去服务端数据读取、缓存和较小 bundle 的优势。

**关键考点：**`'use client'` 是模块边界、props 可序列化、实例生命周期。

**常见追问或变体：**不要把 token 或服务端密钥作为客户端 props；SDK CSS 也要按其文档处理加载时机。

## 11. Node.js：HTTP 流式上传的背压、大小限制与中止

**参考答案：**将请求可读流直接通过 `pipeline()` 接到对象存储可写流，使下游慢时自动反压上游；在转换流计数并超过限制即报错/销毁。请求 `aborted` 或超时应 abort 对象存储上传、销毁所有流并清理 multipart 临时分片。对象 key 使用服务端生成的 UUID，文件名仅作经过净化的展示元数据。

**关键考点：**背压、pipeline 错误传播、取消清理和不可信元数据。

**常见追问或变体：**MIME 类型不能只信客户端 header，必要时检测魔数并做病毒扫描。

## 12. NestJS：API 版本演进与兼容性治理

**参考答案：**公开 API 常用 URI 版本便于路由、缓存和文档；若强调资源语义可用 Header/媒体类型版本。为每个版本维护独立 DTO/serializer 与 OpenAPI，适配层将旧字段明确映射到领域模型；灰度监控旧版本调用量，在公告和 sunset 后下线。仅把字段改成 optional 会让“缺失”和“语义变化”混淆。

**关键考点：**版本选择、契约测试、弃用周期与可观测下线。

**常见追问或变体：**版本不应绕过鉴权和限流，跨版本错误码也应保持可预测。

## 13. 数据库与接口：游标分页的一致性与翻页令牌安全

**参考答案：**按 `created_at DESC, id DESC` 排序，下一页条件为 `created_at < lastCreatedAt OR (created_at = lastCreatedAt AND id < lastId)`，并建立同序复合索引。token 编码最后一条排序键、排序方向、筛选条件摘要和过期时间，再签名或服务端存储，防止客户端改游标跨越租户/筛选边界。

**关键考点：**稳定排序键、seek pagination、复合索引、令牌不可篡改。

**常见追问或变体：**更新排序字段会影响翻页稳定性；强快照需求需在 token 中绑定 snapshot 或使用数据库一致性读。

## 14. AI 前端：生成式 UI 的结构化输出与安全降级

**参考答案：**定义受控组件白名单与 JSON Schema，例如 `RefundStatusCard` 只接受订单展示 id、状态和允许的动作；服务端验证 schema、权限和业务状态后才下发。流式时展示已验证字段或骨架，不把半截 JSON 当 HTML 执行；最终校验失败则展示转义后的纯文本和“暂无法生成卡片”，记录 schema 版本、拒绝原因和请求 trace。

**关键考点：**模型输出不可信、allowlist 渲染、增量与最终校验分层。

**常见追问或变体：**结构化输出不能替代订单权限校验；动作按钮仍须走普通后端鉴权。

## 15. Agent/RAG：多租户检索中的权限过滤与可解释引用

**参考答案：**写入时给每个 chunk 绑定 tenant、文档版本、ACL 和删除状态；检索前由可信后端按当前用户生成强制过滤条件，重排和生成只接收已授权片段。引用返回 chunk id、版本和授权判定以支持审计，但前端只展示当前用户仍可访问的内容。权限变更要使检索缓存和会话记忆失效/降权；历史答案再次打开时重新校验引用。

**关键考点：**ACL 先于向量召回、引用可追溯、缓存与记忆会过期。

**常见追问或变体：**不能把 tenant id 仅交给模型遵守；删除文档也要从索引和缓存链路清除。
