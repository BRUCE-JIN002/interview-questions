# 2026-07-16 全栈偏前端面试题

## 1. JavaScript：`Array.prototype.sort` 的稳定性与比较器契约

- 题型：基础原理 / 场景题
- 难度：中等
- 考察方向：综合整理；ECMAScript 排序语义
- 题目：现代 JavaScript 的 `sort` 是否稳定？比较器应满足哪些约束？若比较器返回值不满足传递性，线上列表可能出现什么问题？

## 2. TypeScript：`unknown`、`any` 与边界数据解析

- 题型：TypeScript / 场景题
- 难度：中等
- 考察方向：[TypeScript Handbook](https://www.typescriptlang.org/docs/handbook/2/narrowing.html)
- 题目：为什么接口响应、`catch` 错误和 `JSON.parse` 结果应优先视为 `unknown`？设计一个运行时校验后返回类型安全用户对象的边界层。

## 3. ES6+：`Map` 的 SameValueZero 键比较与对象键陷阱

- 题型：基础原理
- 难度：中等
- 考察方向：[MDN Map](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Map)
- 题目：`Map` 如何比较键？`NaN`、`-0` 和两个结构相同的对象分别会怎样？何时应使用 `Map` 而非普通对象？

## 4. 浏览器：合成层、`transform` 动画与显存风险

- 题型：性能 / 场景题
- 难度：中等
- 考察方向：综合整理；浏览器渲染流水线
- 题目：为什么通常建议用 `transform` 和 `opacity` 做动画？它是否一定更快？说明图层提升、栅格化、GPU 内存以及如何用 DevTools 验证。

## 5. 网络：HTTPS 页面中的 Mixed Content 与第三方资源治理

- 题型：网络 / 安全
- 难度：中等
- 考察方向：[MDN Mixed content](https://developer.mozilla.org/docs/Web/Security/Mixed_content)
- 题目：HTTPS 页面引入 HTTP 图片、脚本、iframe 时浏览器会如何处理？为第三方资源设计一套上线前检测和运行时降级方案。

## 6. React：受控输入在并发更新下的输入体验

- 题型：框架原理 / 场景题
- 难度：困难
- 考察方向：[React input](https://react.dev/reference/react-dom/components/input)
- 题目：搜索框的 `value` 由父组件控制，同时每次输入都会触发昂贵筛选。如何保证输入不掉帧、结果可延后、旧请求不会覆盖新结果？

## 7. Vue 3：`KeepAlive` 缓存生命周期与资源释放

- 题型：框架原理 / 场景题
- 难度：中等
- 考察方向：[Vue KeepAlive](https://vuejs.org/guide/built-ins/keep-alive.html)
- 题目：解释 `mounted` / `activated` / `deactivated` / `unmounted` 在 `KeepAlive` 下的关系。一个缓存页面含轮询、WebSocket 和表单草稿，应如何管理？

## 8. 手写：实现可组合的 `pipeAsync`

- 题型：手写实现
- 难度：中等
- 考察方向：综合整理；Promise 链式调用
- 题目：实现 `pipeAsync(...fns)`：前一函数的结果传给后一函数，支持同步或异步函数，任一环节异常时停止并以 reject 结束；空函数列表应原样返回输入。

## 9. 算法：合并 K 个升序链表

- 题型：算法
- 难度：困难
- 考察方向：LeetCode 23；堆 / 分治
- 题目：给定 `k` 个已升序链表，合并为一个升序链表。分别说明小顶堆和分治解法的时间、空间复杂度，并给出一种实现。

## 10. Next.js：国际化路由与缓存键设计

- 题型：全栈工程 / 场景设计
- 难度：困难
- 考察方向：[Next.js Internationalization](https://nextjs.org/docs/app/guides/internationalization)
- 题目：App Router 电商站需要支持 `/zh-CN`、`/en-US`，价格还受地区和登录会员等级影响。如何划分 Middleware、路由参数、服务端数据读取与缓存键，避免串语言或串价？

## 11. Node.js：事件循环阻塞与 CPU 密集型接口拆分

- 题型：全栈工程 / 性能排障
- 难度：困难
- 考察方向：[Node.js event loop](https://nodejs.org/en/learn/asynchronous-work/dont-block-the-event-loop)
- 题目：Node BFF 的图片元数据计算和 JSON 压缩导致所有接口 p99 升高。如何确认是事件循环阻塞，并在 `worker_threads`、子进程、队列异步化之间选择？

## 12. NestJS：RBAC 与资源级授权的落点

- 题目：后台有“组织管理员、项目成员、访客”三种角色，成员只能访问自己项目的数据。Guard、业务服务、数据库查询各应承担什么授权职责？如何避免只校验角色导致越权？
