# 2026-07-22 全栈偏前端每日面试参考答案

## 1. JavaScript：Proxy 的不变量与响应式边界

**参考答案：**Proxy 必须维持目标对象不可配置属性、不可扩展性等不变量。例如目标上存在不可配置且不可写的属性，`get` 必须返回其真实值，否则会抛 `TypeError`。响应式系统通常跳过 DOM、Promise、Date/Map 等内建对象、冻结对象和带外部资源句柄的实例；按原始对象缓存 proxy，并允许 `markRaw` 式逃生。

**关键考点：**trap 不是任意拦截器；代理身份稳定、循环引用与副作用对象边界。

**常见追问或变体：**`receiver` 影响 getter 中的 `this`；不要在 `get` 中无条件创建新代理。

## 2. TypeScript：`satisfies` 与配置对象的类型约束

**参考答案：**类型标注会把表达式收窄到标注类型；`as` 可跳过校验且不安全；`satisfies` 校验可赋值性，同时保留推断出的字面量类型。

```ts
type Route = { path: `/${string}`; auth: boolean };
const routes = {
  home: { path: '/', auth: false },
  profile: { path: '/profile', auth: true },
} satisfies Record<string, Route>;
```

**关键考点：**配置防拼写错误、字面量保留与运行时校验仍不可省。

**常见追问或变体：**`satisfies` 不会把值转换成目标类型，不能替代 schema 验证。

## 3. ES6+：`structuredClone`、转移与不可克隆值

**参考答案：**它能克隆循环引用、Map、Set、Date、TypedArray 等；JSON 会丢失 `undefined`、BigInt、类型信息且不能处理循环。函数、DOM 节点、WeakMap/WeakSet 不能 structured clone。transfer `ArrayBuffer` 后发送方 buffer 会被 detach，避免复制，适合大二进制数据；clone 则双方保有独立副本。

**关键考点：**clone 是数据副本，transfer 是所有权移动。

**常见追问或变体：**转移后访问原 buffer 是错误，调用方必须明确其生命周期。

## 4. 事件循环：渲染、微任务与长任务切片

**参考答案：**JS 长 task 未结束前浏览器不能处理绘制，所以先设 loading 再同步计算也看不到更新。每个 task 后会清空微任务队列，持续递归 `queueMicrotask` 同样可能饿死渲染。把工作切成小块，用 `scheduler.postTask`/`setTimeout`/`requestIdleCallback` 在 task 间让出；每块检查 `AbortSignal`，结束时只提交仍属于当前任务的结果。

**关键考点：**微任务优先不等于更流畅；取消需协作式检查。

**常见追问或变体：**CPU 密集且可并行的计算可迁至 Worker，并以消息协议取消。

## 5. 浏览器：跨标签页状态同步与一致性

**参考答案：**同源实时通知优先 BroadcastChannel；`storage` 事件是兼容降级且不会通知写入页；SharedWorker 适合共享连接/协调；权威数据仍由服务端或重新拉取决定。消息携带版本号、操作 id 与时间戳，接收端丢弃旧版本并在缺口时重拉。登出广播后清除内存、持久缓存和敏感 query cache，服务端令牌也必须失效。

**关键考点：**客户端广播不是可靠消息队列，也不是最终一致性来源。

**常见追问或变体：**购物车适合 operation-based 合并；余额等强一致数据必须以服务端结果覆盖。

## 6. 网络：CORS 预检缓存与凭证请求

**参考答案：**凭证请求要求响应精确回显允许的 Origin，并配合 `Access-Control-Allow-Credentials: true`；通配符 Origin 不被允许。非简单方法、header 或 content-type 会触发 OPTIONS 预检，服务端可在可信 Origin 白名单下设置有限 `Access-Control-Max-Age` 来减少预检。CORS 只控制浏览器是否读取响应，不验证用户身份，服务端仍必须鉴权、授权和防 CSRF。

**关键考点：**Origin 白名单、预检缓存与 CORS 安全边界。

**常见追问或变体：**若动态回显 Origin，要发送 `Vary: Origin` 以避免 CDN 缓存串用。

## 7. React：并发渲染下的 Transition 与请求竞争

**参考答案：**Transition 可把非紧急列表更新降级，保持输入响应；它不取消网络请求、不规定响应顺序。每次请求递增序号，回包仅在序号仍最新时提交；同时终止上一次 fetch，并在卸载时 cleanup。

```ts
let latest = 0;
async function search(q: string, signal: AbortSignal) {
  const id = ++latest;
  const data = await fetch(`/api/search?q=${encodeURIComponent(q)}`, { signal }).then(r => r.json());
  if (id === latest) setResults(data);
}
```

**关键考点：**调度优先级与数据竞争是两件事。

**常见追问或变体：**缓存库要把 query key 包含全部筛选条件，避免不同请求共享错误结果。

## 8. Vue 3：`defineModel` 与受控组件契约

**参考答案：**`defineModel('open')` 对应 `open` prop 和 `update:open` event。prop 属于父组件，子组件直接修改会破坏单向数据流。关闭时先进入 `pending`，等待父级或 `beforeClose` 确认成功才 emit `false`；失败则保持开启并反馈错误，避免先关后被父级旧值回写。

**关键考点：**受控值唯一来源、emit 更新和异步状态机。

**常见追问或变体：**修饰符用于定义转换规则，不应承担权限或异步副作用。

## 9. 算法：LRU 缓存

**参考答案：**哈希表从 key 到链表节点，双向链表头表示最近使用、尾表示最久未用。`get` 查表后移到头；`put` 更新已有节点或插入头，超容量时移除尾并删表。容量为 0 时不保留节点。

```ts
class LRUCache {
  private map = new Map<number, number>();
  constructor(private capacity: number) {}
  get(key: number) { const v = this.map.get(key); if (v === undefined) return -1; this.map.delete(key); this.map.set(key, v); return v; }
  put(key: number, value: number) { if (this.map.has(key)) this.map.delete(key); this.map.set(key, value); if (this.map.size > this.capacity) this.map.delete(this.map.keys().next().value!); }
}
```

**关键考点：**标准解是哈希表加双向链表；上例依赖 Map 插入顺序实现同一语义。

**常见追问或变体：**生产缓存还要考虑 TTL、容量估算、并发和淘汰回调。

## 10. Next.js：Server Actions 的授权与重放防护

**参考答案：**Server Action 仍是可被调用的写接口。Action 内先取得 session，再验证输入 schema 和记录所有权；不要信任表单中的 userId。使用框架的同源保护，并为关键写操作加入一次性 nonce/幂等键；将业务错误映射为可展示的稳定结果，成功后 `revalidatePath` 或 `revalidateTag`。数据库事务是最终一致性边界。

**关键考点：**服务端执行不等于可信输入；授权贴近资源查询。

**常见追问或变体：**文件上传和跨域调用应使用专用受控 API，不强塞进普通 Action。

## 11. Node.js：背压与流式代理

**参考答案：**`res.write()` 返回 false 表示下游缓冲已满，必须等待 `drain`，否则慢客户端会导致内存增长。用 `pipeline` 或 async iterator 逐块转发；监听客户端 `close`/`aborted` 后 abort 上游 fetch/stream，并清理 timer。设置超时、最大响应大小和有限并发，代理 SSE 时及时 flush 合规事件。

**关键考点：**下游速度决定读取速度；断连必须向上游传播。

**常见追问或变体：**不要把完整流先聚合成 Buffer 再转发。

## 12. NestJS：多租户数据隔离与事务边界

**参考答案：**从已验证的 token/host 映射得到 tenantId，写入 request-scoped context 或显式传入应用服务；Guard 负责认证和 tenant 解析，Repository 的每个查询强制带 tenant 条件，最好由数据库 RLS 或复合唯一索引兜底。绝不采用客户端 body 的 tenantId 决定权限。跨表写入放同一事务，事务内也必须传递 tenant 作用域。

**关键考点：**租户上下文来自可信身份；应用层过滤需数据库层防线补强。

**常见追问或变体：**后台管理员切租户需要显式 impersonation 审计，不复用普通用户 token。

## 13. 接口设计：幂等写入与异步任务查询

**参考答案：**`POST /imports` 带 `Idempotency-Key`，服务端以租户、用户、key 和请求摘要唯一化，首次创建任务，重复请求返回同一任务或冲突。`GET /imports/:id` 返回 `queued/running/succeeded/partially_failed/failed`、进度、错误摘要和结果链接；失败仅在错误可重试、输入可复用时允许创建新 attempt。前端轮询或订阅状态，不把 202 当成功完成。

**关键考点：**幂等键作用域、状态机、部分失败的可解释结果。

**常见追问或变体：**结果下载 URL 要短期签名并再次校验任务归属。

## 14. AI 前端：生成式 UI 的结构化输出校验

**参考答案：**模型输出不可信，直接 `innerHTML` 会引入 XSS、钓鱼按钮和不可控资源加载。限定组件白名单，要求模型输出 versioned JSON，并在服务端和客户端按 JSON Schema 校验；未知类型或不兼容版本降级为纯文本/人工配置。所有写入、支付、外部跳转按钮由宿主应用生成，展示参数摘要并在执行前按风险确认。

**关键考点：**模型只能提议 UI，渲染器和权限策略才是执行边界。

**常见追问或变体：**日志需记录 schema 版本、校验失败和用户实际批准的操作。

## 15. Agent：长会话上下文压缩与可恢复执行

**参考答案：**原始对话和工具结果存可检索日志；摘要只保留当前目标、已确认事实、未决问题和约束；事实记忆必须带来源、时间与置信度；任务状态保存步骤 DAG、输入版本、输出引用和 checkpoint。恢复时根据 execution id 和每个副作用工具的 idempotency key 查询既有结果，已成功步骤不重放；不确定状态先对账再继续，并向用户展示恢复计划。

**关键考点：**摘要不是事实来源；状态持久化、幂等与对账保证可恢复。

**常见追问或变体：**压缩应可追溯，不能把工具返回中的指令当作可信长期记忆。
