# 2026-07-02 全栈偏前端面试题

## 1. JavaScript `WeakRef` 与 `FinalizationRegistry` 的适用边界

- 题型：八股文 / 场景题
- 难度：高级
- 来源或考察方向：[MDN WeakRef](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/WeakRef)、[MDN FinalizationRegistry](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/FinalizationRegistry)
- 题目正文：图片预览模块希望用 `WeakRef` 缓存大对象，并通过 `FinalizationRegistry` 释放关联资源。为什么不能依赖对象在确定时间被回收？弱引用适合解决什么问题，又不适合替代哪些显式清理机制？请说明闭包或 Map 仍持有强引用时会发生什么。

## 4. HTTP/2 多路复用仍然可能出现的队头阻塞

- 题型：八股文 / 网络
- 难度：高级
- 来源或考察方向：[RFC 9113: HTTP/2](https://www.rfc-editor.org/rfc/rfc9113)、[RFC 9000: QUIC](https://www.rfc-editor.org/rfc/rfc9000)
- 题目正文：HTTP/2 已支持单连接多路复用，为什么 TCP 丢包仍可能让多个请求同时等待？HTTP/3 基于 QUIC 后改善了哪一层的队头阻塞，又没有消除哪些应用层阻塞？线上选择协议时还应考虑哪些兼容性与观测指标？
