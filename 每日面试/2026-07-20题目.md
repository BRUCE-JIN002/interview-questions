# 2026-07-20 全栈偏前端每日面试题

## 1. JavaScript：ES Module 的 live binding、循环依赖与初始化时序

- 题型：原理与输出分析题
- 难度：中高级
- 可能来源或考察方向：[MDN import](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/import)
- 题目：为什么 `import { count } from './store.js'` 不是对 `count` 的值拷贝？两个模块循环 import 时，哪些情况下会触发 TDZ 的 `ReferenceError`？如何拆分共享状态以避免初始化顺序依赖？

## 2. TypeScript：`exactOptionalPropertyTypes` 与 PATCH 接口语义

- 题型：类型设计与接口场景题
- 难度：中高级
- 可能来源或考察方向：综合整理
- 题目：为 `PATCH /users/:id` 设计 TypeScript 入参。如何区分“不更新字段”“显式清空字段”和“字段值为 undefined”？说明 `prop?: T`、`prop: T | undefined`、`exactOptionalPropertyTypes` 与运行时 DTO 校验的边界。

## 3. Promise：thenable 吸收与重复 settle 防护

- 题型：原理与手写题
- 难度：中高级
- 可能来源或考察方向：综合整理（Promises/A+ 方向）
- 题目：`Promise.resolve(thenable)` 为什么不能简单返回 thenable？手写一个最小 `resolvePromise`，处理 getter 抛错、thenable 同时调用 resolve/reject、以及 thenable 返回自身三类情况。

## 4. 浏览器：页面冻结、恢复与后台任务可靠性

- 题型：浏览器生命周期场景题
- 难度：中高级
- 可能来源或考察方向：综合整理
- 题目：一个编辑器页面在切到后台、进入 BFCache 或被系统冻结后，自动保存、轮询和 WebSocket 状态出现错乱。如何组合 `visibilitychange`、`pagehide/pageshow`、`freeze/resume`、心跳重连与版本校验来治理？哪些收尾动作不应依赖 `unload`？

## 5. 网络：`Vary`、CDN 缓存键与内容错配

- 题型：网络与线上排障题
- 难度：中高级
- 可能来源或考察方向：综合整理
- 题目：灰度发布后，部分用户拿到其他语言或其他登录态的页面。解释 `Vary` 如何影响共享缓存；CDN 缓存键应如何处理语言、Cookie、设备和实验桶？为什么不能随意对完整 Cookie 做 `Vary`？

## 6. React：Suspense 瀑布、预加载与边界拆分

- 题型：React 性能场景题
- 难度：中高级
- 可能来源或考察方向：[React Suspense](https://react.dev/reference/react/Suspense)
- 题目：商品详情页中组件各自发起数据请求，导致串行瀑布和多个 loading 闪烁。如何利用路由级预加载、请求去重、Suspense 边界与 `startTransition` 优化首屏和切换体验？边界拆得越细越好吗？

## 7. Vue 3：`computed`、`watch` 与副作用调度

- 题型：框架原理与代码审查题
- 难度：中级
- 可能来源或考察方向：综合整理
- 题目：搜索页把 API 请求写进 `computed`，并在多个 `watch` 中同时修改 query、页码和筛选条件，偶发重复请求与旧结果覆盖。请解释三者职责，并给出可取消、可去重、DOM 读取时序正确的重构方案。

## 8. 手写：支持并发、超时与取消的 `mapLimit`

- 题型：手写题
- 难度：中高级
- 可能来源或考察方向：综合整理
- 题目：实现 `mapLimit(items, limit, mapper, { signal, timeout })`：结果顺序与输入一致；最多 `limit` 个任务运行；外部取消或单任务超时后不再启动新任务，并确保已启动任务收到取消信号。说明失败策略。

## 9. 算法：网格中的最短路径与状态扩展

- 题型：算法题
- 难度：中级
- 可能来源或考察方向：综合整理（BFS 同类）
- 题目：给定 `0/1` 网格，`0` 可走、`1` 为障碍；最多可消除 `k` 个障碍，求从左上到右下的最短步数，不可达返回 `-1`。写出 BFS 解法，并说明为什么 visited 不能只记录坐标。

## 10. Next.js：用户相关数据与缓存隔离

- 题型：全栈工程与安全题
- 难度：中高级
- 可能来源或考察方向：[Next.js 缓存指南](https://nextjs.org/docs/app/guides/caching)
- 题目：App Router 的页面同时有公共商品信息和登录用户的价格/权益。如何划分缓存边界、认证读取位置和缓存键，避免把 A 用户的内容缓存给 B 用户？登录态改变后如何失效并验证？

## 11. Node.js：JWT 密钥轮换与 JWKS 缓存

- 题型：鉴权与线上运行题
- 难度：中高级
- 可能来源或考察方向：综合整理（OIDC/JWKS 方向）
- 题目：BFF 验证第三方 IdP 的 JWT，IdP 开始轮换签名密钥后出现间歇性 401。设计按 `kid` 查询、带 TTL 的 JWKS 缓存、未知 `kid` 刷新、并发合并、失败降级和日志指标；哪些 token 字段必须校验？

## 12. NestJS：多协议应用的认证与异常契约

- 题型：NestJS 架构题
- 难度：中高级
- 可能来源或考察方向：综合整理
- 题目：同一 NestJS 服务同时提供 HTTP、WebSocket 和消息队列消费者。如何复用身份解析、权限判定和领域错误码，却分别适配三种 transport 的上下文、异常映射、重试与死信策略？

## 13. 数据库：避免偏移分页在高频列表中的漏读与重复

- 题型：接口与数据库设计题
- 难度：中高级
- 可能来源或考察方向：综合整理
- 题目：订单按 `created_at DESC` 滚动加载，新增订单后 `OFFSET/LIMIT` 出现重复和漏项。设计一个稳定的 cursor pagination 协议，包含排序键、tie-breaker、索引、下一页 cursor 编码及筛选条件变更策略。

## 14. AI 前端：流式生成的状态一致性与断线恢复

- 题型：AI 前端场景设计题
- 难度：中高级
- 可能来源或考察方向：综合整理
- 题目：对话接口会依次流出文本 token、工具调用状态、结构化卡片和最终消息。前端如何设计事件协议和状态机，处理乱序/重复事件、断线续传、取消、重新生成及页面刷新后的恢复，避免把半成品当最终结果？

## 15. Agent/MCP：工具输出 Schema 校验与不可信内容隔离

- 题型：Agent 安全与工程题
- 难度：高级
- 可能来源或考察方向：[MCP Tools 规范](https://modelcontextprotocol.io/specification/2025-06-18/server/tools)
- 题目：Agent 调用 MCP 工具读取工单和知识库，工具返回的文本可能含恶意指令或不合规字段。客户端应如何利用 `outputSchema`、来源标记、内容分区、最小化展示和高风险操作确认，防止工具结果直接驱动越权行为？
